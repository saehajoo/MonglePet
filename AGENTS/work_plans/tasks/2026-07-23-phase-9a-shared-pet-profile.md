# Phase 9A 공유 권장 프로필

## 상태

- 상태: planned
- 생성일: 2026-07-23
- 마지막 갱신: 2026-07-23

## 목표

- 펫의 애니메이션뿐 아니라 제작자가 의도한 행동·이동 경험도 선택적으로 공유한다.
- 받는 사용자가 공유 설정을 검토하고 적용 여부를 결정하게 한다.
- 설치 식별자, 화면 좌표와 사용 이력을 패키지 밖에 유지한다.

## 선행 조건

- Phase 9에서 이동 모드와 펫별 이동 설정의 Domain 모델·범위를 확정한다.

## 범위

- 기존 `pet.json`과 분리된 선택적·버전 지정 `recommended-profile.json`
- 자동·수동 행동 모드, 수동 선택과 행동 루틴
- 위치 좌표를 제외한 이동 모드, 속도, 거리, 정지 반경, 전면 앱 창 선호와 모드별 이동 중 애니메이션 설정
- 자동 앱 규칙의 별도 포함 선택과 bundle identifier·유휴 시간 검토
- 내보내기 전 공유 프로필 요약·개인정보 검토
- 가져오기에서 `펫만 설치`와 `권장 설정 적용` 선택
- 새 설치와 기존 설치 교체의 서로 다른 적용 정책
- 기존 프로필 없는 `.monglepet`과의 하위 호환

## 제외 범위

- 설치 UUID, 로컬 펫 키와 편집 marker
- 고정 화면 좌표, 디스플레이 식별자, 현재 포인터·창 위치와 이동 이력
- 펫 크기, 클릭 통과, 깨움·재움 같은 현재 앱 상태
- 실행 코드, 스크립트와 플러그인
- 클라우드 동기화, 웹 게시와 다운로드
- 여러 프로필을 항목별로 병합하는 기능

## 검토 결론

- 로컬 `BehaviorProfile`이나 `settings.json`을 그대로 복사하지 않고 공유 전용 DTO를 사용한다.
- 공유 프로필은 받는 사용자의 설정을 강제하는 데이터가 아니라 제작자의 권장 preset이다.
- 내보내기에서는 `펫 설정도 함께 공유`를 명시적으로 선택하고, 가져오기에서는 적용 전 내용을 요약한다.
- 새 설치는 사용자가 적용을 선택한 경우에만 권장 프로필을 편집 가능한 로컬 프로필로 복사한다. 선택하지 않으면 표준 기본 프로필을 만든다.
- 기존 설치 교체는 현재 로컬 프로필을 기본 보존한다. 사용자가 명시적으로 선택할 때만 권장 프로필 전체로 교체하며 초기 버전에는 부분 병합을 제공하지 않는다.
- 자동 규칙은 앱 bundle identifier를 통해 제작자의 사용 앱을 드러낼 수 있고 Windows에서 그대로 호환되지 않으므로 기본 제외하고 별도 선택·검토 항목으로 둔다.
- 자동 모드가 공유되었지만 유효한 자동 규칙이 없으면 해당 펫의 기본 행동 루틴을 재생한다.

## 제안 스키마 윤곽

```json
{
  "schemaVersion": 1,
  "behavior": {
    "mode": "manual",
    "manualSequenceID": "default",
    "sequences": []
  },
  "movement": {
    "mode": "freeRoaming",
    "speed": 120,
    "cursorDistance": 80,
    "stopRadius": 24,
    "prefersFrontmostWindow": true,
    "cursorFollowingMotionID": "run",
    "freeRoamingMotionID": "walk"
  },
  "automaticRules": []
}
```

- 필드 이름과 허용 범위는 Phase 9의 Domain 모델이 확정된 뒤 결정한다.
- 행동 단계가 참조하는 모션 ID와 수동 선택 ID는 해당 펫 패키지 안에서 다시 검증한다.
- 공유 프로필의 미래 schema를 지원하지 못할 때 펫만 설치할지 패키지 전체를 거부할지는 구현 전에 확정한다.

## 작업 순서

- [ ] 1단계: 공유 프로필 Domain·DTO, 값 범위와 참조 무결성 테스트
- [ ] 2단계: 선택적 내보내기, 개인정보 요약과 아카이브 왕복 검증
- [ ] 3단계: 가져오기 요약과 새 설치 적용·미적용 흐름
- [ ] 4단계: 기존 설치 교체 시 로컬 프로필 보존·명시적 전체 교체
- [ ] 5단계: 손상·미래 schema·기존 패키지 호환과 실제 앱 왕복 QA

## 검증 방법

- 행동 모드, 수동 선택, 루틴과 이동 설정의 encode/decode 왕복을 검증한다.
- 존재하지 않는 모션·루틴 참조, 중복 ID와 범위 밖 값을 거부한다.
- 설치 UUID, 화면 좌표, 디스플레이 ID와 현재 상태가 아카이브에 들어가지 않는지 검사한다.
- 자동 규칙을 제외·포함한 두 내보내기 경로와 검토 요약을 확인한다.
- 새 설치의 `펫만 설치`·`권장 설정 적용`과 기존 설치의 기본 보존을 각각 확인한다.
- `recommended-profile.json`이 없는 기존 `.monglepet`을 이전과 동일하게 가져온다.

## 남은 위험 / 후속 작업

- 루틴과 자동 규칙을 부분 병합하면 ID 충돌과 참조 변경이 생기므로 초기 버전은 전체 보존 또는 전체 적용만 제공한다.

- 미래 공유 프로필 schema를 만났을 때 펫만 안전하게 설치하는 정책을 패키지 보안 검증과 함께 결정해야 한다.
