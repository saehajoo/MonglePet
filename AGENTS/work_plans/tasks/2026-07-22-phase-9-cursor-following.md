# Phase 9 펫 이동 모드와 상호작용

## 상태

- 상태: in_progress
- 생성일: 2026-07-22
- 마지막 갱신: 2026-07-23

## 목표

- 사용자가 위치 고정, 마우스 따라가기와 자유 이동 중 원하는 방식을 선택하게 한다.
- 자유 이동에서는 선택적으로 현재 전면 앱의 대표 창 근처를 선호하게 한다.
- 이동을 행동 결정과 분리하고 작업 방해·상시 성능 비용·위치 기록을 피한다.

## 범위

- `fixed`, `cursorFollowing`, `freeRoaming` 이동 모드와 펫별 이동 설정
- 포인터·자유 이동 목표 좌표와 부드러운 창 위치 보간
- 자유 이동의 안전 영역, 목표 대기 시간과 무작위 목표 선택
- 전면 앱 PID와 화면상 대표 창 위치를 이용한 선택적 창 주변 목표
- 대표 창을 찾지 못하거나 전체 화면인 경우 `visibleFrame` 안 목표로 복구
- 사용자 드래그 우선권과 이동 일시 중지
- 다중 모니터, visible frame과 화면 경계 보정
- 재우기·잠금·절전·동작 줄이기 생명주기
- 이동 애니메이션 선택과 기존 행동 복귀
- 마우스 따라가기·자유 이동별 선택 애니메이션과 참조 이름 변경·삭제 복구
- 쓰다듬기 일회성 반응 연결

## 제외 범위

- 포인터 이동 이력 저장과 분석
- 전면 앱 창의 제목·내용·이동 이력 저장
- 접근성 권한을 사용하는 정확한 focused-window 추적
- 충돌, 중력과 화면 가장자리 보행 물리
- 여러 펫 군집 이동

## 결정사항

- 기존 드래그 위치를 유지하는 `fixed`가 기본값이다.
- 이동 모드는 행동의 자동·수동 모드와 독립적이다.
- 이동 애니메이션은 자동 이동으로 창 위치가 실제 변하는 동안에만 적용하고 사용자 드래그에는 적용하지 않는다.
- 일회성 상호작용은 이동 애니메이션보다 우선하며, 이동이 끝나면 기존 행동 루틴의 진행 위치로 복귀한다. 이동 애니메이션 미지정·누락 시 기존 행동을 계속 표시한다.
- 비활성 상태에는 포인터 polling이나 이동 timer를 유지하지 않는다.
- 포인터와 전면 앱 창 위치는 현재 목표 계산에만 사용하고 저장하지 않는다.
- 자유 이동은 별도 접근성 권한 없이 얻을 수 있는 대표 창을 최선 노력으로 사용하고 실패 시 안전한 화면 목표로 복구한다.
- 공유 가능한 이동 설정은 펫별 프로필에 저장하되 고정 화면 좌표와 자동 이동 중 좌표는 공유 프로필에서 제외한다.
- 드래그, 재우기, 잠금·절전과 동작 줄이기가 자동 이동보다 우선한다.

## 검증 방법

- 세 이동 모드의 전환, 속도, 정지 반경과 화면 경계의 결정적 좌표 테스트를 실행한다.
- 대표 창 있음·없음, 전체 화면, 여러 창·화면과 좌표계 변환을 가짜 provider로 검증한다.
- 이동 시작·도착의 상태 전환, 짧은 정지 히스테리시스와 모드별 애니메이션 선택·미지정 폴백을 검증한다.
- 드래그 시작·종료, 재우기·깨우기와 잠금·절전 중 timer 상태를 확인한다.
- 다중 모니터 사이 이동과 디스플레이 분리 후 복구를 확인한다.
- 기능 비활성 상태 CPU와 timer 기준선을 측정한다.
- 포인터·창 위치나 이력이 설정·로그 파일에 저장되지 않는지 확인한다.

## 남은 위험 / 후속 작업

- 이동 애니메이션이 없는 펫은 현재 기본 애니메이션으로 안전하게 동작해야 한다.
- Window Server에서 대표 창을 얻지 못하는 앱과 전체 화면 환경에서는 화면 안전 영역 폴백의 체감 품질을 실제 앱에서 확인해야 한다.
- Core Graphics의 화면 좌표와 AppKit 창 좌표가 서로 다른 기준을 사용하므로 다중 모니터 좌표 변환을 단위 테스트로 고정해야 한다.
- 정확한 focused-window 추적이 실제 사용에서 꼭 필요하다고 확인될 때만 접근성 권한을 요구하는 후속 기능을 별도로 검토한다.
