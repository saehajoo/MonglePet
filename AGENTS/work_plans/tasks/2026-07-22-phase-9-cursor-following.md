# Phase 9 마우스 따라가기와 이동 상호작용

## 상태

- 상태: planned
- 생성일: 2026-07-22
- 마지막 갱신: 2026-07-22

## 목표

- 사용자가 원할 때 펫이 포인터를 부드럽게 따라가도록 한다.
- 이동을 행동 결정과 분리하고 작업 방해·상시 성능 비용·위치 기록을 피한다.

## 범위

- 따라가기 활성화, 이동 속도, 커서 거리와 정지 반경
- 포인터 목표 좌표와 부드러운 창 위치 보간
- 사용자 드래그 우선권과 이동 일시 중지
- 다중 모니터, visible frame과 화면 경계 보정
- 재우기·잠금·절전·동작 줄이기 생명주기
- 이동 애니메이션 선택과 기존 행동 복귀
- 쓰다듬기 일회성 반응 연결

## 제외 범위

- 포인터 이동 이력 저장과 분석
- 충돌, 중력과 화면 가장자리 보행 물리
- 여러 펫 군집 이동

## 열린 질문

- 없음

## 결정사항

- 기능 기본값은 꺼짐이다.
- 비활성 상태에는 포인터 polling이나 이동 timer를 유지하지 않는다.
- 포인터 위치는 현재 목표 계산에만 사용하고 저장하지 않는다.
- 드래그, 재우기, 잠금·절전과 동작 줄이기가 자동 이동보다 우선한다.

## 작업 순서

- [ ] 1단계: 순수 이동 계산 모델과 가짜 좌표 테스트
- [ ] 2단계: PetMovementController와 저전력 scheduler
- [ ] 3단계: Overlay 생명주기·드래그·다중 화면 연결
- [ ] 4단계: 설정 저장과 UI
- [ ] 5단계: 이동 애니메이션·쓰다듬기와 실제 앱 QA

## 검증 방법

- 속도, 정지 반경과 화면 경계의 결정적 좌표 테스트를 실행한다.
- 드래그 시작·종료, 재우기·깨우기와 잠금·절전 중 timer 상태를 확인한다.
- 다중 모니터 사이 이동과 디스플레이 분리 후 복구를 확인한다.
- 기능 비활성 상태 CPU와 timer 기준선을 측정한다.
- 포인터 위치나 이력이 설정·로그 파일에 저장되지 않는지 확인한다.
