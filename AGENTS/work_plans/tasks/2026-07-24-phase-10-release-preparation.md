# Phase 10 공개 준비와 이동 성능 보완

## 상태

- 상태: in_progress
- 생성일: 2026-07-24
- 마지막 갱신: 2026-07-24

## 목표

- 로그인 시 MonglePet 자동 실행을 사용자가 일반 설정에서 제어할 수 있게 한다.
- 처음 사용하는 사람도 설치와 주요 기능을 이해할 수 있는 README와 비상업적 소스 공개 라이선스를 준비한다.
- Apple Developer Program 가입 전에는 미서명·미공증 Preview ZIP으로 제한된 테스트 배포 기반을 만든다.
- 마지막 단계에서 마우스 따라가기와 자유 이동의 Release CPU만 짧게 측정하고 필요한 경우에만 최적화한다.

## 범위

- `SMAppService.mainApp`의 실제 등록 상태를 단일 원본으로 사용하는 로그인 실행 설정
- 초기 미등록·등록 기록 없음, 활성, 사용자 승인 필요와 실제 등록 실패 UI
- `PolyForm Noncommercial License 1.0.0`, 사용자 중심 `README.md`와 제3자 고지
- 회사 맥에서 재현 가능한 Release·Preview ZIP 생성 절차와 미확인 개발자 앱 실행 안내
- Release 독립 실행 환경에서 마우스 따라가기·자유 이동의 짧은 CPU 측정
- 지속적인 고CPU가 확인된 경우에만 Time Profiler 기반 이동 경로 최적화

## 제외 범위

- 1시간 메모리 증가 검사
- Developer ID Application 서명과 Apple notarization
- 정식 안정 버전 전체 릴리스 회귀
- GitHub 최신 버전 조회와 자동 업데이트
- DMG·PKG 설치 패키지
- 모든 앱 상태를 포함한 장시간 성능 기준선
- 회사 맥에서 최종 배포 ZIP과 SHA-256 체크섬 생성

## 열린 질문

- 코드 저작권 표기에 사용할 개인 또는 프로젝트 명칭은 라이선스 파일 작성 단계에서 확정한다.

## 결정사항

- 작업 순서는 로그인 실행, 문서·라이선스, Preview 배포, 짧은 이동 성능 측정, 조건부 최적화 순서다.
- 로그인 실행 여부는 `settings.json`에 저장하지 않고 `SMAppService.status`를 실제 상태의 단일 원본으로 사용한다.
- 승인 필요 상태는 등록 요청을 유지한 채 시스템 로그인 항목 설정으로 이동하는 복구 버튼을 제공한다.
- 미서명·미공증 앱은 정식 안정 버전이 아닌 Preview로 표시하며 Gatekeeper 비활성화나 quarantine 제거 명령을 안내하지 않는다.
- SHA-256은 코드 서명이 아니라 최종 배포 파일의 무결성 체크섬이다. 파일이 달라지면 값도 달라지므로 최종 Preview ZIP과 체크섬은 개인 맥에서 함께 생성한다.
- 회사 맥에서는 배포 절차와 검증 명령만 준비하고 실제 배포 파일 생성은 별도 개인 맥 작업 계획의 최우선 후속 작업으로 진행한다.
- 마우스 따라가기 2분, 정지 1분, 자유 이동 2분, 대기 1분만 Release 독립 실행 환경에서 측정한다.
- 이동 중 CPU 10% 이상이 지속되거나 정지·대기 중 1~2% 이상이 지속될 때만 최적화를 진행한다.

## 작업 순서

- [ ] 1단계: 로그인 시 자동 실행 서비스, 일반 설정 UI와 검증
- [ ] 2단계: 사용자 중심 README, 비상업적 코드 라이선스와 제3자 고지
- [ ] 3단계: 개인 맥에서 재현할 수 있는 미서명·미공증 Preview Release ZIP과 체크섬 생성 절차
- [ ] 4단계: 마우스 따라가기·자유 이동 Release 단기 성능 측정
- [ ] 5단계: 측정 결과 기준을 넘은 경우에만 이동 성능 최적화

## 검증 방법

- 가짜 로그인 서비스로 미등록·활성·승인 필요·찾을 수 없음 상태와 등록·해제 실패를 단위 테스트한다.
- 설정 화면에서 로그인 자동 실행 토글, 상태 설명과 승인 설정 버튼의 접근성 노출을 UI 테스트한다.
- 실제 앱에서 켜기·끄기, 승인 필요 안내와 완전 종료 후 시스템 로그인 항목 상태 복원을 확인한다.
- README 링크, 라이선스 범위, ZIPFoundation MIT 고지와 펫 자산 라이선스 분리를 확인한다.
- 개인 맥 작업에서 생성한 Preview ZIP을 새 임시 사용자 환경에서 압축 해제하고 Apple이 제공하는 개인정보 보호 및 보안 승인 경로로 실행한다.
- Release 빌드를 Xcode와 분리해 실행하고 지정한 이동 상태의 CPU 평균·최댓값과 기기 환경을 기록한다.
- 코드 변경이 생기면 관련 단위 테스트부터 전체 XCTest, XCUITest와 Debug·Release 빌드로 확장한다.

## 진행 로그

- 2026-07-24: 1시간 메모리 검사와 전체 릴리스 회귀를 현재 범위에서 제외하고 기존 다중 모니터·Space QA를 재사용하기로 했다.
- 2026-07-24: Developer ID 서명·공증은 Apple Developer Program 가입 이후 후속 작업으로 분리하고 현재는 Preview 배포만 준비하기로 했다.
- 2026-07-24: 성능 측정은 마지막에 마우스 따라가기와 자유 이동만 짧게 확인하고 기준을 넘을 때만 최적화하기로 했다.
- 2026-07-24: `SMAppService.mainApp` adapter와 상태 기반 일반 설정 UI를 구현하고 별도 설정 JSON에는 자동 실행 여부를 추가하지 않았다.
- 2026-07-24: 로그인 실행 단위 테스트 6개, 설정 화면 대상 XCUITest 1개, 전체 단위 XCTest 303개를 통과하고 Debug 빌드 성공을 확인했다.
- 2026-07-24: 1단계는 실제 macOS 로그인 항목 켜기·끄기와 재실행 수동 QA를 남겨 두었다.
- 2026-07-24: 최종 Preview ZIP과 SHA-256 체크섬은 회사 맥에서 만들지 않고 별도 개인 맥 작업의 최우선 후속 단계로 분리했다.
- 2026-07-24: 실제 Xcode 실행에서 아직 등록 기록이 없는 앱 경로가 `.notFound`로 표시되어 토글이 잠기는 문제를 확인했다. 이 상태도 첫 등록이 가능한 꺼짐 상태로 안내하고 `register()`를 호출하도록 수정했다.
- 2026-07-24: 보완 후 로그인 실행 대상 테스트 6개와 전체 단위 XCTest 307개가 통과했고 선택적 로컬 WebP fixture 1개만 건너뛰었다. 실제 켜기·끄기 재검증은 계속 남겨 두었다.

## 남은 위험 / 후속 작업

- 미공증 Preview 앱은 Apple이 개발자와 변조 여부를 확인할 수 없으므로 일반 사용자용 정식 배포로 취급하지 않는다.
- Apple Developer Program 가입 후 Developer ID 서명, Hardened Runtime 최종 검증과 notarization을 별도 작업으로 진행한다.
- 기능 범위가 확정되면 개인 맥에서 최종 Preview ZIP과 SHA-256 체크섬을 생성·검증하는 별도 계획을 가장 먼저 진행한다.
- 1시간 메모리 검사, 공증된 배포 파일 전체 회귀, 자동 업데이트와 DMG는 후속 안정화 작업으로 유지한다.
