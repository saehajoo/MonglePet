# 자동 규칙 앱 선택기

## 상태

- 상태: in_progress
- 생성일: 2026-07-24
- 마지막 갱신: 2026-07-24

## 목표

- 사용자가 Bundle Identifier를 알지 못해도 실행 중인 앱이나 직접 선택한 `.app`으로 앱별 자동 규칙을 만들고 수정할 수 있게 한다.
- 현재 Bundle Identifier 기반 행동 결정과 저장 스키마는 유지한다.

## 범위

- 실행 중인 일반 앱의 아이콘·이름·Bundle Identifier 목록
- MonglePet 자체, 중복 Identifier와 Identifier가 없는 프로세스 제외
- 표준 파일 선택창을 통한 `.app` 선택과 Bundle Identifier 추출
- 앱 이름과 Identifier를 함께 보여주는 선택 결과
- 선택 결과와 세 선택 경로를 하나로 정리한 앱 선택 UI
- 별도 팝오버에서 제공하는 Bundle Identifier 직접 입력
- 자동 규칙 우선순위·입력 없음 시간의 좌우 증감과 직접 숫자 입력
- 등록된 규칙의 요약 카드와 펼칠 수 있는 세부 편집 UI
- 일반 탭 하단의 앱 실행·앱 정보 순서 정리
- 앱 규칙 추가와 기존 앱 규칙 변경 흐름

## 제외 범위

- 전체 디스크 또는 설치 앱 디렉터리 자동 스캔
- 앱 경로·아이콘·실행 이력의 영구 저장
- 접근성 권한 요청
- 브라우저 URL, 창 제목이나 문서명에 따른 규칙
- 설정 스키마 변경

## 열린 질문

- 없음

## 결정사항

- 실행 중인 앱은 공개 `NSWorkspace.runningApplications` 정보만 사용한다.
- 설치 앱은 App Sandbox 범위를 넓히지 않고 사용자가 표준 파일 선택창에서 `.app`을 직접 고른다.
- 영구 설정에는 기존과 같이 Bundle Identifier만 저장한다.
- 앱 선택기는 현재 선택 결과를 먼저 보여주고 `대상 앱 선택…` 메뉴 하나에서 `열려 있는 앱`, `설치된 앱 파일 선택…`, `Bundle Identifier 직접 입력…`을 제공한다.
- 직접 입력은 설치되지 않은 앱과 특수 사례를 위한 고급 경로로 두고, 별도 팝오버의 둥근 테두리 입력 필드와 명시적인 적용 버튼을 사용한다.
- `idleAtLeast` 도메인·저장 명칭은 호환성을 위해 유지하되 사용자 화면에는 `입력 없음 규칙`으로 표시한다.
- 등록된 규칙은 활성 상태, 조건, 행동 루틴과 우선순위를 한 줄로 요약하고 수정할 때만 세부 설정을 펼친다.
- 자동 규칙의 정수 조절은 macOS 기본 상하 `Stepper` 대신 좌우 버튼과 직접 입력 가능한 가운데 숫자 필드를 사용한다.
- 일반 탭의 앱 관련 보조 정보는 마지막에 `앱 실행`, `앱 정보` 순서로 둔다.

## 검증 방법

- 실행 앱 후보의 필터링·중복 제거·정렬을 순수 입력으로 단위 테스트한다.
- 임시 `.app` fixture에서 이름과 Bundle Identifier 추출, 잘못된 파일 거부를 확인한다.
- 자동 규칙 탭에서 실행 중 앱 선택·앱 파일 선택·직접 입력 경로의 접근성 노출을 확인한다.
- 선택한 Identifier가 기존 설정 저장과 행동 규칙에 그대로 전달되는지 확인한다.
- 실제 앱에서 실행 중 앱과 `/Applications`의 앱을 각각 선택해 규칙 추가·수정과 재실행 복원을 확인한다.
