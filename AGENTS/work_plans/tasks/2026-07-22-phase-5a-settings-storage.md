## 목표

- 사용자 설정을 버전이 지정된 JSON 파일로 안전하게 저장하고 복원한다.
- Domain 모델과 저장 DTO를 분리해 `Duration`, enum과 자동 규칙 조건을 명시적인 공개 스키마로 변환한다.
- 손상·부분 오류·미래 버전 설정이 있어도 원본을 보존하고 앱이 안전한 기본값으로 실행되게 한다.

## 범위

- `AppSettings`, `OverlaySettings` Domain 모델
- schema version 1 저장 DTO와 명시적 문자열/discriminator 인코딩
- 행동 단계 밀리초, 재생 속도와 자동 규칙의 Domain 변환
- 항목 단위 검증·복구 결과
- 5MiB 설정 파일 상한
- 같은 디렉터리 임시 파일, synchronize와 원자적 교체 저장
- 손상 파일 격리와 미래 버전 원본 보존·쓰기 차단
- 임시 디렉터리 및 JSON fixture 기반 단위 테스트

## 제외 범위

- 설정 SwiftUI 화면과 편집 상태
- 앱 실행 시 overlay 위치·표시 상태 적용
- 펫 라이브러리 설치 ID의 실제 존재 여부 확인
- 로그인 시 자동 실행 등록
- 설정 스키마 버전 2 이상의 실제 마이그레이션

## 결정사항

- overlay 너비는 96–384pt, 기본값은 192pt다.
- 행동 재생 속도는 0.25–4.0배다.
- 행동 목록 최대 100개, 목록별 단계 최대 100개, 자동 규칙 최대 100개로 제한한다.
- 행동 단계와 유휴 조건의 최대 시간은 24시간이다.
- 로그인 시 실행은 JSON에 중복 저장하지 않고 향후 `SMAppService` 상태를 원본으로 사용한다.
- 알 수 없는 자동 규칙 조건은 type을 보존하되 규칙을 비활성화한다.

## 검증 방법

- 기본 설정을 저장하고 다시 읽어 Domain 값이 같은지 확인한다.
- JSON에 정수 밀리초, 명시적 enum 문자열과 조건 discriminator가 기록되는지 확인한다.
- 잘못된 enum, UUID, 크기, 속도, 목록 참조와 알 수 없는 조건을 항목 단위로 복구한다.
- 손상 JSON은 별도 파일로 이동하고 원본을 덮어쓰지 않는지 확인한다.
- 미래 schema version은 원본을 유지하고 저장을 거부하는지 확인한다.
- 저장 실패와 교체 후 임시 파일이 남지 않는지 확인한다.
- 전체 `MonglePetTests`와 Debug 빌드를 실행한다.
