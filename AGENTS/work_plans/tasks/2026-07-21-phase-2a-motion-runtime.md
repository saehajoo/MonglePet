# Phase 2A 모션 모델과 내장 펫 재생

## 상태

- 상태: completed
- 생성일: 2026-07-21
- 마지막 갱신: 2026-07-21
- 현재 하위 단계: 완료

## 목표

- 운영체제 API와 분리된 펫·모션·프레임 모델을 만든다.
- 프레임별 지속 시간을 지키는 테스트 가능한 재생 시간축을 구현한다.
- 임시 내장 펫 모션을 현재 non-activating 오버레이에서 반복 재생한다.
- 펫을 재우면 재생 시계를 멈추고 깨우면 안전하게 다시 시작한다.

## 범위

- `PetDefinition`, `PetMotion`, `MotionFrame` 순수 Swift 모델
- atlas와 프레임 사각형을 표현하는 프레임 참조 모델
- 반복·비반복 모션의 프레임 시간축
- 프레임 변경만 전달하는 런타임 재생기
- AppKit/Core Animation 기반 atlas 프레임 표시
- 임시 내장 펫 정의와 모션 자산
- 표시 상태와 프레임 재생 생명주기 연결

## 제외 범위

- `.monglepet` JSON 파싱, 검증과 설치
- 외부 PNG·WebP·GIF·APNG 가져오기
- Codex v1·v2 atlas adapter
- 자동·수동 행동 선택과 여러 모션 스케줄링
- 최종 몽글이 캐릭터 디자인
- 설정 저장과 재실행 복원

## 열린 질문

- 없음

## 결정사항

- Domain 모델은 AppKit, SwiftUI와 CoreGraphics를 import하지 않는다.
- 런타임 시간은 `Duration`을 사용하고 저장 DTO의 정수 밀리초 변환은 패키지 로더 단계에서 담당한다.
- 프레임 좌표는 패키지 규격과 같은 왼쪽 위 원점의 정수 픽셀 사각형으로 표현한다.
- 프레임 선택 규칙은 순수 시간축으로 분리해 실제 타이머 대기 없이 단위 테스트한다.
- 화면 렌더러는 현재 프레임이 바뀔 때만 Core Animation layer 내용을 갱신한다.
- 2A 내장 자산은 런타임 검증용이며 최종 캐릭터 자산으로 간주하지 않는다.

## 진행 로그

- 2026-07-21: Phase 1 완료 후 Phase 2A 계획 작성 및 착수.
- 2026-07-21: `PetDefinition`, `PetMotion`, `MotionFrame`과 픽셀 좌표 모델을 운영체제 API 없는 `nonisolated` 값 타입으로 구현.
- 2026-07-21: 반복·비반복 경계를 계산하는 순수 `FrameTimeline`과 주입 가능한 스케줄러 기반 `FramePlayer` 구현.
