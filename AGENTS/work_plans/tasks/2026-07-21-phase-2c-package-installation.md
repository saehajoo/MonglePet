# Phase 2C 외부 패키지 설치

## 상태

- 상태: completed
- 생성일: 2026-07-21
- 마지막 갱신: 2026-07-21
- 현재 하위 단계: 완료

## 목표

- 사용자 선택 `.monglepet` ZIP 파일을 App Sandbox 범위 안에서 안전하게 읽는다.
- 압축 엔트리를 추출 전에 검사하고 제한된 임시 공간에만 해제한다.
- 2B 로더로 전체 재검증한 패키지만 앱 라이브러리에 원자적으로 설치한다.
- 같은 패키지 ID의 중복 설치, 별도 사본과 명시적 교체를 안전하게 처리한다.

## 범위

- ZIPFoundation 0.9.20 정확한 버전 Swift Package 의존성
- 사용자 선택 URL의 security-scoped 접근 수명 관리
- 압축 파일 20 MiB, 해제 크기 100 MiB와 최대 압축률 제한
- ZIP 엔트리 경로 탈출, 절대 경로, 중복과 심볼릭 링크 사전 거부
- 임시 디렉터리 개별 추출과 2B `PetPackageLoader` 전체 검증
- Application Support `MonglePet/Library/<installation-uuid>` 설치
- 신규·별도 사본 설치와 같은 패키지의 명시적 원자 교체
- 실패 시 staging 및 임시 파일 정리
- 실제 ZIP fixture와 임시 라이브러리 기반 통합 단위 테스트

## 제외 범위

- `NSOpenPanel`과 설정 화면의 펫 추가 UI
- 설치 전 미리보기 및 사용자 확인 화면
- 설치된 펫 선택, 삭제와 라이브러리 관리 UI
- Codex, GIF, APNG와 PNG 시퀀스 importer
- 현재 화면의 펫을 설치된 펫으로 교체하는 동작
- 앱 업데이트용 패키지 서명 및 원격 다운로드

## 열린 질문

- 없음

## 결정사항

- ZIPFoundation은 외부 프로세스 없이 동작하는 인프로세스 ZIP 구현으로 사용하고 0.9.20에 고정한다.
- ZIPFoundation의 전체 해제 편의 API 대신 엔트리를 먼저 전부 검사한 뒤 개별 추출한다.
- 엔트리는 파일과 디렉터리만 허용하고 심볼릭 링크 및 특수 항목은 전부 거부한다.
- 절대 경로, 역슬래시, 빈 구성요소, `.`, `..`를 거부하고 Unicode 정규화 및 대소문자를 무시한 중복도 금지한다.
- 압축 파일은 20 MiB, 전체 해제 데이터는 100 MiB, 엔트리 수는 2,000개, 압축률은 엔트리별·전체 100:1을 상한으로 둔다.
- security-scoped 접근은 원본 검사와 추출 동안만 유지하며 설치 후 bookmark를 보존하지 않는다.
- 신규 설치와 별도 사본은 UUID 디렉터리를 만들고, 교체는 같은 패키지 ID가 확인된 설치 ID에만 허용한다.
- staging은 최종 라이브러리와 같은 디렉터리에 만들고 검증 후 rename/replace하여 최종 경로에 부분 설치가 보이지 않게 한다.

## 검증 방법

- ZIPFoundation으로 정상 `.monglepet` ZIP fixture 생성 후 설치 및 재로딩
- 경로 탈출, 절대 경로, 심볼릭 링크와 대소문자 중복 엔트리 거부 테스트
- 압축 크기, 해제 크기, 엔트리 수와 압축률 상한 테스트
- 손상 ZIP, 유효하지 않은 manifest와 이미지에서 설치 디렉터리가 남지 않는지 확인
- 중복 거부, 별도 UUID 설치와 같은 패키지 ID 교체 테스트
- security-scoped 접근 시작·종료 균형 테스트
- 전체 `MonglePetTests`와 Debug 빌드

## 완료 결과

- `.monglepet` ZIP을 엔트리 사전 검사 후 제한된 임시 디렉터리에 안전하게 해제한다.
- 2B 로더로 전체 재검증한 패키지만 Application Support 라이브러리에 UUID로 설치한다.
- 중복 패키지는 기본 거부하고 호출자가 별도 사본 또는 같은 ID 교체를 명시할 수 있다.
- staging과 backup을 이용해 신규 설치와 교체를 원자적으로 처리하고 실패 흔적을 정리한다.
- App Sandbox의 사용자 선택 파일 접근은 처리 중에만 유지하고 원본 bookmark는 저장하지 않는다.
