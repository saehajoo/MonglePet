# 사용자 자유 행동 기본값 전환

## 상태

- 상태: completed
- 생성일: 2026-07-22
- 마지막 갱신: 2026-07-22

## 목표

- 첫 실행에는 삭제할 수 없는 `기본` 행동 루틴 하나만 제공한다.
- 집중·휴식·수면이라는 시스템 고정 의미와 기본 자동 규칙을 제거한다.
- 사용자가 펫 애니메이션, 행동 루틴과 자동 규칙의 의미와 우선순위를 자유롭게 구성하게 한다.
- 설정이 비었거나 참조 애니메이션이 현재 펫에 없더라도 현재 선택된 펫의 기본 애니메이션을 안전하게 재생한다.

## 범위

- 현재 펫 기본 애니메이션을 나타내는 예약 행동 단계 참조
- 삭제 불가능하지만 편집 가능한 시스템 `기본` 행동 루틴 하나
- 초기 자동 규칙 제거와 조건 종류에 무관한 사용자 priority 평가
- 내장 몽글이의 임시 애니메이션을 기본 하나로 축소
- 기존 미수정 4개 프리셋·2개 자동 규칙의 손실 없는 전환
- 행동 설정 UI 명칭과 관련 단위 테스트·문서 갱신

## 제외 범위

- 행동 루틴 이름 변경
- 펫별로 분리된 행동 설정
- 쓰다듬기 상호작용
- 자동 규칙 조건 조합

## 열린 질문

- 없음

## 결정사항

- 행동 루틴이 없다는 이유로 선택 펫을 몽글이로 바꾸지 않는다. 펫 패키지가 누락되거나 손상된 경우에만 내장 몽글이로 복구한다.
- 시스템 `기본` 루틴은 예약 ID를 사용하고 삭제할 수 없지만 단계·시간·속도·반복은 편집할 수 있다.
- `현재 펫의 기본 애니메이션` 예약 참조를 행동 단계에서 명시적으로 제공한다.
- 새 행동 루틴의 첫 단계도 현재 펫 기본 애니메이션 참조로 시작한다.
- 자동 규칙은 enabled, 큰 priority, 배열 순서 기준으로 평가하고 조건 종류별 내장 우선순위를 두지 않는다.
- 기존 설정이 정확히 종전 기본 프리셋과 규칙인 경우에만 새 기본 하나로 전환한다. 수정된 데이터는 사용자 설정으로 보존한다.

## 작업 순서

- [x] 3단계: 설정 정규화·기존 프리셋 전환
- [x] 4단계: 설정 UI와 내장 몽글이 애니메이션 정리
- [ ] 5단계: 테스트, Debug 빌드와 실제 앱 QA

## 검증 방법

- 빈 설정을 로드하면 기본 루틴 하나와 자동 규칙 0개가 실행 중 준비되는지 확인한다.
- 기본 루틴과 새 사용자 루틴의 첫 단계가 현재 펫 기본 애니메이션을 참조하는지 확인한다.
- 기본 루틴은 삭제할 수 없고 custom 루틴 삭제 시 참조가 정리되는지 확인한다.
- 앱 규칙과 유휴 규칙이 동시에 일치할 때 조건 종류가 아니라 priority와 배열 순서로 선택되는지 확인한다.
- 기존 미수정 4개 프리셋은 새 기본 하나로 전환되고 수정된 기존 설정은 보존되는지 확인한다.
- `idle`이 없는 사용자 펫에서도 기본 루틴과 누락 애니메이션 참조가 펫 기본 애니메이션으로 재생되는지 확인한다.
- 관련 단위 테스트, 전체 단위 테스트와 Debug 빌드를 실행한다.
