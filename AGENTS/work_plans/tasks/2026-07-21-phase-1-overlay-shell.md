# Phase 1 앱 셸과 펫 오버레이

## 상태

- 상태: in_progress
- 생성일: 2026-07-21
- 마지막 갱신: 2026-07-21
- 현재 하위 단계: 1B 펫 오버레이 완료, 1C 기본 상호작용 착수 전

## 목표

- Dock 아이콘 없이 메뉴 막대에서 실행되는 macOS 앱 셸을 만든다.
- 현재 작업 앱을 방해하지 않는 투명 펫 오버레이의 기반을 단계적으로 구현한다.
- 펫을 이동하고 숨겼다가 메뉴 막대에서 다시 표시할 수 있는 최소 사용자 흐름을 완성한다.

## 범위

- 1A: AppKit 앱 생명주기, `AppCoordinator`, `NSStatusItem`, 설정창과 종료
- 1B: non-activating `NSPanel`, 임시 투명 PNG와 Space 동작
- 1C: 드래그, 깨우기·재우기와 화면 밖 위치 보정
- 1D: 단위 테스트, UI 스모크 테스트, 수동 QA와 초기 성능 기준선

## 제외 범위

- WebP 및 프레임 애니메이션 재생
- `.monglepet` 패키지 로딩과 설치
- 자동·수동 행동 엔진
- 설정 JSON 영구 저장
- 클릭 통과 사용자 설정
- 정식 몽글이 캐릭터 디자인

## 열린 질문

- 다중 모니터 수동 QA는 실제 연결 가능한 디스플레이 환경에서 수행한다.

## 결정사항

- 첫 실행 상태는 `awake`로 한다.
- 초기 펫 크기는 192×208pt로 한다.
- 초기 위치는 주 디스플레이 오른쪽 아래에서 32pt 안쪽으로 한다.
- 앱 실행 시 설정창을 자동으로 열지 않는다.
- 설정창을 닫아도 앱은 메뉴 막대에서 계속 실행한다.
- 1A 메뉴는 설정 열기와 앱 종료만 제공한다.
- 깨우기와 재우기는 펫 오버레이가 존재하는 1C에서 추가한다.

- `xcodebuild` Debug 빌드
- `MonglePetTests` 단위 테스트

- 앱 실행 시 일반 창과 Dock 아이콘이 자동으로 나타나지 않는지 수동 확인
- 설정창을 닫은 뒤 메뉴 막대에서 다시 열 수 있는지 확인

## 진행 로그

- 2026-07-21: Phase 1 계획 작성 및 1A 앱 셸 착수.
- 2026-07-21: `NSStatusItem`은 agent 앱의 자체 XCUI 접근성 트리에 노출되지 않아 메뉴 구조와 액션은 단위 테스트, 설정창 표시는 UI 테스트로 분리.
- 2026-07-21: 스토리보드 없는 AppKit 앱에서 진입점이 delegate를 명시적으로 생성·연결하도록 구성하고 `AppCoordinator`를 시작하도록 수정.
- 2026-07-21: `LSUIElement`은 Debug·Release 모두 활성화하고 메뉴 막대의 설정·종료 액션, 재사용 가능한 설정창을 구현.
- 2026-07-21: Debug 빌드, 단위 테스트 4개, 설정창 UI 스모크 테스트 통과. UI 테스트에서는 agent 앱 창을 자동화할 수 있도록 테스트 실행 인자에서만 일반 활성화 정책을 사용.
- 2026-07-21: 프로젝트 전용 임시 몽글이 이미지를 생성하고 크로마키 제거 후 알파 PNG로 Asset Catalog에 추가. 정식 캐릭터 자산이 아니라 오버레이 검증용으로만 사용.
- 2026-07-21: `PetWindow`, `PetWindowController`, `PetOverlayView`를 Overlay 계층에 추가. 192×208pt, 화면 오른쪽 아래 32pt, `.floating`, `.canJoinAllSpaces`, `.fullScreenAuxiliary`와 non-activating 구성을 적용.
- 2026-07-21: `AppCoordinator`가 시작 시 펫 오버레이를 표시하도록 연결. 패널 구성, 위치, 번들 이미지 로딩, 표시 및 키 윈도우 비점유를 포함한 단위 테스트 7개 통과.
- 2026-07-21: 오버레이 UI 스모크 테스트를 추가했으나 Xcode 테스트 worker 생성 단계가 두 차례 멈춰 자동 실행은 완료하지 못함. 빌드된 실제 `LSUIElement` 앱 프로세스를 실행해 수동 화면 확인을 진행.
- 2026-07-21: 실제 `LSUIElement` 앱에서 투명 펫 오버레이 표시와 다른 앱 조작 비방해를 사용자 확인으로 검증하고 1B 완료 처리.

## 완료 결과

- 1A 앱 셸과 1B 펫 오버레이 구현 완료. Phase 1 전체는 진행 중.

## 남은 위험 / 후속 작업

- 메뉴 막대 UI 자동화는 macOS 접근성 환경에 따라 수동 검증이 추가로 필요할 수 있다.
- `LSUIElement` 앱의 메뉴 막대 클릭, 설정창 활성화와 종료 동작은 실제 실행 환경에서 수동 확인을 유지한다.
- 오버레이 UI 테스트는 Xcode UI 테스트 worker가 정상화된 뒤 다시 실행한다.
- 모든 Space, 전체 화면과 다중 모니터 동작은 1D 수동 QA에서 최종 확인한다.
