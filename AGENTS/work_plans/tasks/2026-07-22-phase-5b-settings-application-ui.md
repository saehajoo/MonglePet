## 결정사항

- 첫 실행이나 손상 파일 복구 시 저장 좌표 `0, 0`을 적용하지 않고 기존 우하단 기본 배치를 사용한다.
- 저장된 디스플레이가 있으면 해당 visible frame에 위치를 보정하고, 없으면 현재 화면 중 가장 적합한 화면으로 복구한다.
- 크기는 조작 중 즉시 적용하되 슬라이더 조작이 끝날 때 한 번 저장한다.
- 클릭 통과 상태에서도 메뉴 막대의 설정 진입과 깨우기·재우기 복구 경로를 유지한다.
- 미래 스키마가 감지되면 파일을 보존하고 영구 설정 편집을 비활성화하되 메뉴 막대의 일시적인 깨우기·재우기는 허용한다.

## 검증 방법

- 기본값, 정상 파일, 항목 복구, 손상 파일과 미래 버전의 설정 세션 상태를 확인한다.
- 크기 변경이 종횡비를 유지하고 위치가 visible frame 안으로 보정되는지 확인한다.
- 클릭 통과가 `NSPanel.ignoresMouseEvents`에 적용되는지 확인한다.
- 저장된 `tuckedAway`는 숨김 상태로 시작하고 `awake`는 저장 위치에서 시작하는지 확인한다.
- 드래그 후 화면 식별자와 좌표가 설정 파일에 저장되는지 확인한다.
- 설정창의 상태, 모드, 크기와 클릭 통과 컨트롤이 노출되고 변경되는지 확인한다.
- 전체 `MonglePetTests`, 관련 UI 테스트와 Debug 빌드를 실행한다.
