# Phase 8 로컬 펫 공유

## 상태

- 상태: planned
- 생성일: 2026-07-22
- 마지막 갱신: 2026-07-22

## 목표

- 사용자가 만든 펫을 표준 `.monglepet` 파일로 내보내고 다른 사용자가 가져올 수 있게 한다.
- 향후 웹 카탈로그도 같은 데이터 전용 패키지를 사용할 수 있는 기반을 만든다.

## 범위

- ZIP 기반 `.monglepet` 내보내기
- 메타데이터, 미리보기, manifest와 참조 이미지 포함
- 편집 marker와 설치·행동 로컬 데이터 제외
- 내보내기 전 전체 검증과 저장 패널
- 가져오기 중복·교체·별도 사본 UX 정리
- 라이선스 확인과 공유 불가 상태 안내

## 제외 범위

- 웹 업로드·다운로드
- 계정, 서명, 평점과 자동 업데이트
- 행동 루틴과 자동 규칙 공유

## 열린 질문

- 없음

## 결정사항

- 공유 형식은 기존 `.monglepet` 규격을 사용하며 별도 설정 파일 형식을 만들지 않는다.
- 앱 bundle identifier가 포함될 수 있는 자동 규칙은 내보내지 않는다.
- 공유받은 패키지는 읽기 전용이며 수정은 편집 가능한 사본에서 수행한다.
- 내보내기 전에 사용자가 라이선스와 제작자 정보를 확인해야 한다.

## 작업 순서

- [ ] 1단계: 내보내기 아카이브 작성기와 보안 제한
- [ ] 2단계: 라이선스·메타데이터 확인 흐름
- [ ] 3단계: 저장 패널과 일반 설정 내보내기 UI
- [ ] 4단계: 가져오기·중복 UX 통합
- [ ] 5단계: 왕복·손상·보안 테스트와 실제 사용자 교환 QA

## 검증 방법

- 내보낸 파일을 새 라이브러리에 가져와 메타데이터와 모든 프레임이 일치하는지 확인한다.
- marker, 설치 UUID와 행동 설정이 아카이브에 없는지 확인한다.
- 경로 탈출, 심볼릭 링크, 과대 파일과 불필요한 엔트리가 내보내기·가져오기에서 거부되는지 확인한다.
- 같은 패키지 ID의 업데이트와 별도 사본 흐름을 확인한다.
