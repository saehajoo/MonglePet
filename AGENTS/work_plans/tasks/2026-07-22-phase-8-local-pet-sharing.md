# Phase 8 로컬 펫 공유

## 상태

- 상태: completed
- 생성일: 2026-07-22
- 마지막 갱신: 2026-07-23

## 목표

- 사용자가 만든 펫을 표준 `.monglepet` 파일로 내보내고 다른 사용자가 가져올 수 있게 한다.
- 향후 웹 카탈로그도 같은 데이터 전용 패키지를 사용할 수 있는 기반을 만든다.

## 범위

- ZIP 기반 `.monglepet` 내보내기
- 메타데이터, 미리보기, manifest와 참조 이미지 포함
- 편집 marker와 설치·행동 로컬 데이터 제외
- 내보내기 전 전체 검증과 저장 패널
- 가져오기 중복·교체·별도 사본 UX 정리
- 라이선스 확인과 공유 불가 상태 안내

## 제외 범위

- 웹 업로드·다운로드
- 계정, 서명, 평점과 자동 업데이트
- 행동 루틴과 자동 규칙 공유

## 열린 질문

- 없음

## 결정사항

- 공유 형식은 기존 `.monglepet` 규격을 사용하며 별도 설정 파일 형식을 만들지 않는다.
- 앱 bundle identifier가 포함될 수 있는 자동 규칙은 내보내지 않는다.
- 공유받은 패키지는 읽기 전용이며 수정은 편집 가능한 사본에서 수행한다.
- 내보내기 전에 사용자가 라이선스와 제작자 정보를 확인해야 한다.

## 작업 순서

- [x] 1단계: 내보내기 아카이브 작성기와 보안 제한
- [x] 2단계: 라이선스·메타데이터 확인 흐름
- [x] 3단계: 저장 패널과 일반 설정 내보내기 UI
- [x] 4단계: 가져오기·중복 UX 통합
- [x] 5단계: 왕복·손상·보안 테스트와 실제 사용자 교환 QA

## 검증 방법

- 내보낸 파일을 새 라이브러리에 가져와 메타데이터와 모든 프레임이 일치하는지 확인한다.
- marker, 설치 UUID와 행동 설정이 아카이브에 없는지 확인한다.
- 경로 탈출, 심볼릭 링크, 과대 파일과 불필요한 엔트리가 내보내기·가져오기에서 거부되는지 확인한다.
- 같은 패키지 ID의 업데이트와 별도 사본 흐름을 확인한다.

## 진행 로그

- 2026-07-22: 사용자 요구와 D-025를 기준으로 계획 작성.
- 2026-07-23: Phase 7 완료 후 1단계 내보내기 아카이브 작성기와 보안 제한 구현을 시작했다.
- 2026-07-23: 설치 패키지를 재검증하고 정규화한 `pet.json`과 참조 이미지 자산만 선별해 ZIP으로 만든 뒤, 가져오기 왕복 검증과 20 MiB 제한을 통과한 결과만 원자적으로 저장하는 `PetPackageExporter`를 추가했다.
- 2026-07-23: 편집 marker·사용하지 않는 파일·알 수 없는 manifest 필드 제외, 정상 왕복, 원본 재검증 실패 시 기존 파일 보존, 잘못된 확장자와 압축 크기 제한 테스트 4개가 통과했다.
- 2026-07-23: Debug 빌드와 전체 단위 XCTest 169개가 통과했고 선택형 로컬 Codex fixture 1개는 구성되지 않아 건너뛰었다.
- 2026-07-23: 제작자·버전·라이선스 검토와 명시적 공유 권한 확인을 내보내기 전에 강제하는 `PetPackageSharingService`를 추가했다. 검토 후 메타데이터가 바뀌면 기존 확인을 무효화한다.
- 2026-07-23: 개인 용도 전용, `All Rights Reserved`, `Unknown`·미지정 라이선스는 확인으로 우회할 수 없게 차단하고 그 밖의 라이선스는 사용자 확인 후 진행하도록 D-029에 기록했다.
- 2026-07-23: 라이선스 정규화·차단, 확인 필수, 차단 우회 방지와 메타데이터 변경 재검토를 포함한 내보내기 테스트 8개가 통과했다.
- 2026-07-23: Debug 빌드와 전체 단위 XCTest 173개가 통과했고 선택형 로컬 Codex fixture 1개는 구성되지 않아 건너뛰었다.
- 2026-07-23: 일반 설정에 설치 펫 전용 내보내기 버튼, 제작자·버전·라이선스 검토 시트, 공유 권한 확인과 macOS 저장 패널을 연결했다. 내장 몽글이는 내보내기 대상에서 제외했다.
- 2026-07-23: 차단된 사용자 펫은 정보 수정으로, 차단된 읽기 전용 펫은 편집 가능한 사본 만들기로 이어지게 했으며 저장 성공과 오류를 설정 화면에 표시한다.
- 2026-07-23: 내장 펫 제외, 선택 설치 펫 검토, 확인된 내보내기 전달과 오류 상태를 포함한 `PetLibrarySession` 테스트 4개를 추가했고 관련 내보내기·세션 테스트가 통과했다.
- 2026-07-23: Debug 빌드와 전체 단위 XCTest 177개가 통과했고 선택형 로컬 Codex fixture 1개는 구성되지 않아 건너뛰었다. 실제 저장 패널 흐름과 내보낸 파일 재가져오기는 Xcode 수동 QA로 남겼다.
- 2026-07-23: 실제 QA에서 검토 시트가 닫히는 중 동기 `NSSavePanel.runModal()`이 시작되어 AppKit breakpoint와 앱 교착이 발생했다. SwiftUI 시트의 `onDismiss` 완료 후 후속 동작을 실행하고 저장 패널은 비동기 `begin`으로 표시하도록 수정했으며 Debug 빌드가 통과했다.
- 2026-07-23: 비동기 AppKit 패널에서도 같은 환경의 `NSSavePanel` 생성 breakpoint가 재현되어 직접 패널 경로를 제거했다. 검증된 임시 아카이브를 메모리 `FileDocument`로 전환하고 SwiftUI `fileExporter`가 대화상자와 최종 쓰기를 담당하도록 교체했으며 Debug 빌드가 통과했다.
- 2026-07-23: 멈춘 실제 프로세스 스택에서 `REPORT_APP_ENTITLEMENTS_INSUFFICIENT → AppKitBreakInDebugger`를 확인했다. App Sandbox의 `ENABLE_USER_SELECTED_FILES`가 `readonly`였던 것이 근본 원인이므로 Debug·Release를 `readwrite`로 변경하고 접근 범위를 사용자 선택 URL로 제한하는 D-030을 기록했다.
- 2026-07-23: 서명 Debug 앱을 별도 빌드하고 `codesign` 결과에 `com.apple.security.app-sandbox`와 `com.apple.security.files.user-selected.read-write`가 함께 포함되는 것을 확인했다.
- 2026-07-23: 새 entitlement로 다시 빌드한 실제 앱에서 공유 권한 확인, 저장 위치 선택과 `.monglepet` 저장이 정상 동작하는 것을 사용자 QA로 확인했다.
- 2026-07-23: 중복 가져오기를 단순 경고에서 비교·선택 시트로 교체했다. 가져올 이름·버전·제작자·패키지 ID와 같은 ID의 모든 설치 후보를 보여주며 현재 선택, 편집 가능 여부와 교체할 설치 UUID를 사용자가 명시적으로 고르게 했다.
- 2026-07-23: `새 설치로 추가`는 새 설치 UUID와 독립 행동 프로필을 갖는 읽기 전용 설치로, `선택 항목 교체`는 선택 UUID의 행동 프로필을 유지하면서 패키지 파일과 편집 가능 상태를 읽기 전용 가져오기 결과로 바꾸는 동작으로 구분했다.
- 2026-07-23: 중복 오류가 가져올 전체 메타데이터를 전달하도록 확장하고 현재 선택 우선 후보 정렬, 명시적 교체 대상과 편집 가능 상태를 검증하는 세션·설치 테스트가 통과했다.
- 2026-07-23: 전체 단위 XCTest 177개와 Debug 빌드가 통과했고 선택형 로컬 Codex fixture 1개는 구성되지 않아 건너뛰었다. 중복 비교·선택 시트의 실제 표시와 두 설치 방식은 Xcode 수동 QA로 남겼다.
- 2026-07-23: 실제 앱에서 중복 패키지 비교 화면, 새 설치 추가, 다중 설치 후보 선택과 명시적 기존 항목 교체가 정상 동작하는 것을 사용자 QA로 확인했다.
- 2026-07-23: 공유 권한 확인 후 생성한 `.monglepet`을 완전히 새 라이브러리에 설치하는 전체 왕복 테스트를 추가했다. 새 설치 UUID, 메타데이터·애니메이션·이미지 일치, canonical 파일 목록, 읽기 전용 상태와 원본 편집 marker 보존을 확인했다.
- 2026-07-23: 공유 manifest에서 설치 UUID·행동 프로필·자동 규칙이 제외되는지 명시적으로 검증했다. 기존 설치 테스트가 손상 ZIP, 경로 탈출, 심볼릭 링크, 경로 충돌, 크기·엔트리·압축률 제한과 실패 격리를 모두 다루는 것을 재확인했다.
- 2026-07-23: Debug 빌드와 전체 단위 XCTest 178개가 통과했고 선택형 로컬 Codex fixture 1개는 구성되지 않아 건너뛰었다. 앞선 실제 앱 내보내기·재가져오기와 중복 처리 QA 결과를 합쳐 Phase 8을 완료했다.
