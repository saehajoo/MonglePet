# Phase 5E 펫 라이브러리와 실행 중 펫 교체

## 상태

- 상태: completed
- 생성일: 2026-07-22
- 마지막 갱신: 2026-07-22
- 현재 하위 단계: 5E-1~5E-5·5E-7 구현 후 사용자 펫 제작 실제 앱 QA 대기

## 목표

- 내장 몽글이와 사용자가 설치한 펫을 하나의 라이브러리에서 확인하고 선택할 수 있게 한다.
- 선택한 펫의 애니메이션 리소스와 행동 런타임을 앱 재실행 없이 안전하게 교체한다.
- `.monglepet`, Codex PNG/WebP atlas, GIF/APNG와 PNG 시퀀스를 가져오고 설치 항목을 관리한다.
- 앱 안에서 PNG 한 장 또는 여러 장으로 편집 가능한 사용자 펫과 펫 애니메이션을 만든다.
- 선택한 설치 항목을 저장하고, 누락되거나 손상된 항목은 내장 몽글이로 안전하게 복구한다.

## 범위

- 내장 펫과 설치된 펫을 나타내는 카탈로그 모델 및 라이브러리 세션
- `selectedPetInstallationID`를 이용한 선택 저장·재실행 복원·누락 항목 폴백
- 여러 atlas를 포함한 설치 펫의 프레임 표시와 실행 중 펫 전환
- 현재 펫의 `PetDefinition`을 설정 화면과 행동 런타임에 동기화
- `.monglepet` 패키지 설치와 중복 패키지 처리
- 사용자 펫 생성, PNG 프레임 순서·간격·반복 설정과 기존 사용자 펫에 애니메이션 추가
- Codex 디렉터리·PNG/WebP spritesheet 및 GIF/APNG·PNG 시퀀스 변환 후 설치
- 설치된 펫 삭제와 현재 선택 항목 삭제 시 내장 펫 복귀
- 단위 테스트, UI 테스트, Debug 빌드와 실제 앱 수동 QA

- 펫별로 분리된 행동 루틴·자동 규칙 설정
- 외부에서 가져온 패키지를 직접 수정하는 범용 패키지 편집기와 수동 spritesheet 좌표 도구
- Windows 라이브러리 구현
- 펫 여러 마리의 동시 표시
- 쓰다듬기 상호작용

## 결정사항

- `selectedPetInstallationID == nil`은 삭제할 수 없는 내장 몽글이를 의미한다.
- 설치 UUID는 라이브러리 항목 식별과 선택 저장에 사용하며 패키지 자체의 `metadata.id`와 구분한다.
- 저장된 설치 UUID가 없거나 패키지가 손상되면 내장 몽글이로 폴백하고 유효하지 않은 선택을 저장소에서도 정리한다.
- 행동 루틴과 자동 규칙은 이번 단계에서도 전역 설정으로 유지한다. 현재 펫에 없는 애니메이션 ID는 보존하고 runtime에서 해당 펫의 선언된 기본 애니메이션으로 폴백한다.
- 펫을 바꾸면 현재 재생을 중단하고 새 펫의 기본 애니메이션에서 시작한 뒤 현재 모드에 맞는 행동을 다시 평가한다.
- 창 너비는 유지하고 새 펫의 기본 애니메이션 첫 프레임 비율에 맞춰 창 높이를 조정한다.
- 설치된 펫의 모든 atlas를 미리 등록하고 프레임의 `atlasID`에 따라 표시 이미지를 교체한다.
- 가져오기 변환 결과는 임시 위치에서 검증한 뒤 기존 `PetLibraryStore`의 원자적 설치 흐름으로 라이브러리에 복사한다.
- MonglePet에서 만든 사용자 펫에는 전용 JSON marker를 넣어 편집 가능 여부를 구분하고 일반 가져오기에서는 외부 marker를 제거한다. 애니메이션 추가는 설치 원본을 직접 변경하지 않고 임시 사본을 전체 검증한 뒤 같은 설치 UUID로 원자적 교체한다.
- PNG 한 장은 정지 애니메이션, 여러 장은 UI가 확정한 순서의 애니메이션으로 만들며 서로 다른 크기는 가장 큰 투명 셀 중앙에 맞춘다.
- 펫 애니메이션의 프레임 간격·반복과 행동 루틴 단계의 유지 시간·재생 속도는 서로 다른 책임으로 유지한다.
- 내장 몽글이는 라이브러리 목록의 첫 항목으로 고정하며 설치 펫은 표시 이름과 설치 UUID 순서로 안정적으로 정렬한다.

## 검증 방법

- 설치 항목을 선택한 뒤 앱을 재실행해 같은 설치 UUID와 애니메이션이 복원되는지 확인한다.
- 저장된 설치 UUID가 없거나 해당 패키지가 손상된 경우 내장 몽글이로 복구되는지 확인한다.
- 서로 다른 크기와 여러 atlas를 가진 펫으로 전환했을 때 프레임, 창 비율과 행동 목록이 함께 갱신되는지 확인한다.
- 현재 펫에 없는 애니메이션을 참조하는 전역 행동 루틴이 펫의 선언된 기본 애니메이션 폴백으로 계속 실행되는지 확인한다.
- PNG 한 장으로 만든 펫이 정지 애니메이션으로 표시되고 여러 PNG가 지정한 순서와 간격으로 재생되는지 확인한다.
- 사용자 펫에 새 애니메이션을 추가하면 같은 설치 항목을 유지하면서 행동 루틴의 펫 애니메이션 목록에 즉시 나타나는지 확인한다.
- 가져온 패키지와 내장 펫에는 애니메이션 추가 버튼이 나타나지 않는지 확인한다.
- 지원하는 각 입력 형식을 가져와 설치·선택·재실행 복원할 수 있는지 확인한다.
- 중복 설치 거부, 별도 설치와 같은 패키지 교체가 의도한 설치 UUID 규칙을 따르는지 확인한다.
- 선택 중인 펫을 삭제하면 내장 몽글이로 전환되고 디스크 항목과 저장 선택이 정리되는지 확인한다.
- 전체 단위 테스트, UI 테스트, Debug 빌드와 실제 앱 수동 QA를 실행한다.
