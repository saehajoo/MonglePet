- 선택 모니터 식별자와 visible frame 기준 정규화 사용자 영역의 로컬 저장·복원
- 기본 펫 투명도와 클릭 통과 중 마우스 겹침 투명도

- `pet.json`의 선택적 `compatibility.createdWithMonglePetVersion`과 `minimumMonglePetVersion`
- 최소 버전 미충족 설치 차단, 제작 앱만 더 새로운 경우 경고 후 설치, 필드가 없는 기존 패키지 허용

## 결정사항

- 다중 모니터 정체는 애니메이션 재생이 아니라 목적지만 대상 화면으로 보정하고 중간 경로를 화면 합집합에 맞추지 않는 이동 경로 문제로 다룬다.
- 화면이 연결된 경우 펫 전체가 통과할 수 있는 경계 지점을 경유하고, 논리 화면 사이에 빈 간격이 있으면 기존 화면 경계까지 이동한 뒤 대상 화면 진입점으로 한 번만 전환한다.
- 대상 화면 진입점을 적용한 뒤 실제 패널 위치가 요청 위치와 일치하지 않으면 대상 화면의 안전한 최종 목표를 적용하고 다음 tick에서 경로를 다시 계산한다.
- 이동 범위와 투명도는 펫 자산이 아니라 현재 기기의 전역 overlay 설정이다. `.monglepet`과 `recommended-profile.json`에 포함하지 않는다.
- 이동 범위 기본값은 `전체 디스플레이`다. 선택 모니터와 사용자 지정 영역은 디스플레이 UUID를 사용하고, 사용자 영역은 해당 화면 visible frame 기준 `0...1` 정규화 사각형으로 저장한다.
- 저장된 모니터를 찾지 못하면 현재 사용 가능한 화면 전체로 안전하게 복구하되 원래 선택 정보는 자동 삭제하지 않는다.
- 위치 고정에는 자동 이동 범위를 적용하지 않는다. 마우스 따라가기는 범위 밖 포인터의 목표를 범위 경계로 제한하고, 자유 이동은 범위 안에서만 목표를 만든다.
- 기본 투명도는 `0.10...1.00`, 마우스 겹침 투명도는 `0.05...1.00`으로 제한한다. 겹침 상태의 실제 투명도는 두 값 중 작은 값이며 기본값은 각각 `1.00`, `0.20`이다.
- 마우스 겹침 투명화는 클릭 통과, awake, 화면 사용 가능 상태에서만 낮은 빈도로 포인터를 확인하고 비활성 상태에는 timer를 유지하지 않는다.
- 겹침 판정은 패널 전체가 아니라 현재 프레임의 알파가 보이는 영역을 사용한다. 구현 복잡도나 성능 문제가 확인되면 캐시한 저해상도 알파 마스크를 사용한다.
- 현재 개발 앱 버전은 `0.1.0`, 빌드 번호는 `1`로 시작한다. 사용자에게는 `MonglePet 0.1.0 (1)` 형태로 표시하고 Git 태그와 Release는 `v0.1.0` 형식을 사용한다.
- 펫 자체 `version`과 MonglePet 앱 버전을 분리한다. 앱 호환 정보는 `pet.json.compatibility` 선택 객체로 두고 생성·내보내기 시 앱이 자동 기록한다.
- 현재 앱이 `minimumMonglePetVersion`보다 낮으면 설치를 차단한다. `createdWithMonglePetVersion`만 더 높고 필수 최소 버전을 충족하면 경고 후 설치를 허용한다.
- 호환 정보가 없는 기존 패키지는 지금과 동일하게 허용한다. 지원하지 않는 패키지 `formatVersion`은 계속 차단하고, 지원하지 않는 선택 권장 프로필은 펫만 설치한다.

## schema-v4 구조

현재 설정 스키마는 v4다. v3 이하 파일은 다음 overlay 필드의 기본값을 추가해 순차 마이그레이션한다.

```json
{
  "schemaVersion": 4,
  "overlay": {
    "screenIdentifier": null,
    "originX": 0,
    "originY": 0,
    "width": 192,
    "clickThrough": false,
    "opacity": 1.0,
    "pointerOverlapFadeEnabled": false,
    "pointerOverlapOpacity": 0.2,
    "movementBoundary": {
      "mode": "allDisplays",
      "screenIdentifier": null,
      "normalizedRect": null
    }
  }
}
```

사용자 지정 영역 예시:

```json
{
  "mode": "customArea",
  "screenIdentifier": "display-example",
  "normalizedRect": {
    "x": 0.1,
    "y": 0.1,
    "width": 0.8,
    "height": 0.8
  }
}
```

펫 패키지 호환 정보 예시:

```json
{
  "formatVersion": 1,
  "version": "1.2.0",
  "compatibility": {
    "createdWithMonglePetVersion": "0.1.0",
    "minimumMonglePetVersion": "0.1.0"
  }
}
```

## 검증 방법

- 좌·우·위·아래 배치, 높이가 다른 화면, 음수 좌표, visible frame 사이 간격과 펫보다 좁은 연결부를 순수 좌표 테스트로 검증한다.
- 낮은 이동 속도에서도 다른 모니터로 진입하고, 요청한 진입점이 적용되지 않을 때 안전한 최종 목표로 복구하는지 가짜 panel adapter로 확인한다.
- 전체 디스플레이·선택 모니터·사용자 지정 영역에서 마우스 따라가기와 자유 이동 목표가 범위를 벗어나지 않는지 확인한다.
- 디스플레이 분리·재연결, 해상도와 visible frame 변경에서 정규화 영역 복원과 안전 폴백을 확인한다.
- schema-v3 fixture를 v4 기본 표시 설정으로 마이그레이션하고 v4 저장 왕복, 범위 밖 투명도·영역 값의 항목 단위 복구와 미래 스키마 보호를 확인한다.
- 클릭 통과가 꺼짐, 펫 숨김, 잠금·절전, 동작 줄이기와 기능 비활성 상태에서 겹침 감지 timer가 남지 않는지 확인한다.
- 현재 프레임의 투명 픽셀과 불투명 픽셀에서 겹침 판정이 다르고, 겹침 해제 후 기본 투명도로 복귀하는지 확인한다.
- 앱 버전의 SemVer 비교, 누락·손상 호환 필드, 최소 버전 미충족 차단과 제작 버전 경고를 단위 테스트한다.
- 호환 필드가 없는 기존 `.monglepet`, 선택 권장 프로필 포함·미포함 패키지와 새 호환 필드의 내보내기·가져오기 왕복을 검증한다.
- 전체 단위 테스트와 Debug 빌드 후 실제 다중 모니터, 클릭 통과와 앱 완전 종료·재실행을 수동 확인한다.

## 남은 위험 / 후속 작업

- 모니터 사이에 펫 전체가 통과할 연결 영역이 없으면 짧은 화면 전환이 필요하며 자연스러운 연출을 실제 기기에서 확인해야 한다.
- 프레임 알파 기반 겹침 판정은 atlas 접근 비용을 만들 수 있으므로 캐시와 비활성 상태 timer 해제를 성능 측정에 포함한다.
- 자동 업데이트를 추가할 때는 GitHub API 직접 다운로드보다 서명된 업데이트 경로와 앱 배포 서명을 먼저 확정해야 한다.
