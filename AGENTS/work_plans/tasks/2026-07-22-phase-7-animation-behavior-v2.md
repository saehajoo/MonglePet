# Phase 7 애니메이션·행동 모델 v2와 펫별 설정

## 상태

- 상태: in_progress
- 생성일: 2026-07-22
- 마지막 갱신: 2026-07-22

## 목표

- PNG 프레임별 재생 시간을 사용자가 직접 편집한다.
- 행동 단계의 초 단위 유지 시간과 재생 속도를 제거하고 애니메이션 사이클로 진행한다.
- 모든 행동 설정을 내장 펫 또는 설치 UUID별로 분리한다.

## 범위

- 프레임별 `durationMs`, 순서, 추가·제거와 미리보기
- `BehaviorStep`의 `motionID`, `repeatCount` 모델
- 완전한 애니메이션 사이클 경계 스케줄러
- `BehaviorProfile`과 내장·설치 펫 키
- 펫별 모드, 수동 선택, 루틴과 자동 규칙 UI
- schema-v2 저장 DTO와 v1 순차 마이그레이션
- 패키지 교체·별도 사본·삭제에 따른 프로필 생명주기

## 제외 범위

- 행동 프로필의 `.monglepet` 포함 또는 사용자 간 공유
- 조건 조합형 자동 규칙
- 마우스 따라가기

## 열린 질문

- 없음

## 결정사항

- 애니메이션 속도의 단일 원본은 프레임별 `durationMs`다.
- 행동 단계 기본 반복 횟수는 1이며 루틴 반복은 `BehaviorSequence.repeats`가 담당한다.
- 같은 패키지 ID의 별도 설치도 설치 UUID가 다르면 독립 프로필이다.
- v1 전역 행동 설정은 당시 선택 펫의 첫 v2 프로필로 이관한다.
- v1 유지 시간은 선택 펫의 실제 사이클 시간으로 가장 가까운 반복 횟수를 계산한다.

## 작업 순서

- [x] 1단계: schema-v2 fixture와 v1 마이그레이션 계약
- [x] 2단계: 프레임별 사용자 애니메이션 편집
- [x] 3단계: 사이클 기반 BehaviorStep·scheduler
- [x] 4단계: BehaviorProfile Domain·저장·세션
- [ ] 5단계: 펫 선택·교체·삭제 생명주기 연결
- [ ] 6단계: 펫별 설정 UI와 전체 회귀 테스트

## 검증 방법

- 서로 다른 프레임 시간을 저장·재로드하고 실제 경계에서 재생하는지 확인한다.
- 여러 애니메이션 단계가 사이클과 반복 횟수대로 이동하는지 가짜 시계로 확인한다.
- 펫 전환 후 각 프로필의 모드·루틴·규칙이 독립적으로 복원되는지 확인한다.
- 같은 설치 교체는 프로필을 유지하고 별도 사본은 공유하지 않는지 확인한다.
- v1 fixture가 데이터 손상 없이 v2로 원자적으로 전환되는지 확인한다.

## 진행 로그

- 2026-07-22: 사용자 요구, D-023과 D-024를 기준으로 계획 작성.
- 2026-07-22: schema-v2 저장 DTO와 v1 마이그레이션 fixture 계약 구현을 시작했다.
- 2026-07-22: 내장 펫 예약 키와 설치 UUID를 명시적 discriminator로 저장하는 schema-v2 DTO를 추가했다.
- 2026-07-22: v1 유지 시간을 선택 펫의 패키지 프레임 사이클에 가장 가까운 반복 횟수로 변환하고, 없는 애니메이션은 현재 펫 기본값 1회로 복구하는 fixture 계약을 고정했다.
- 2026-07-22: schema-v2 마이그레이션 테스트 4개, 관련 설정 테스트 23개와 전체 단위 XCTest 152개가 통과했다. 실제 저장소는 후속 Domain·런타임 전환 전까지 schema-v1을 유지한다.
- 2026-07-22: 편집 가능한 사용자 펫의 기존 프레임 유지·삭제·순서 변경, 새 PNG 추가와 프레임별 16~60000ms 간격 편집 UI를 추가했다.
- 2026-07-22: 편집 미리보기를 현재 초안 순서와 시간으로 재생하고, 저장 시 전용 atlas를 재구성해 전체 패키지 검증 후 같은 설치 UUID로 교체하도록 구현했다. 공유 atlas는 다른 모션이 참조하는 동안 보존한다.
- 2026-07-22: PNG마다 다른 원본 크기로 저장 후 캐릭터가 작아지는 문제를 막기 위해 알파 영역 기반 공통 캔버스 자동 배치와 프레임별 배율·가로·세로 보정을 추가했다. 등록 화면에는 `512×512 px` 투명 PNG와 동일한 캔버스·캐릭터 크기를 권장하도록 안내한다.
- 2026-07-22: 편집 미리보기를 투명 격자 기반 선택 프레임 고정 보기와 전체 재생·일시정지로 나눴다. 새 펫·애니메이션 등록은 기존 초안을 지우는 다시 선택 대신 PNG 누적 추가와 프레임별 간격·배치 편집을 제공하고, 내용 스크롤과 고정 하단 버튼으로 작은 화면의 오버플로우를 방지한다.
- 2026-07-22: Debug 빌드, 전체 단위 XCTest 155개와 기존 XCUITest 5개가 통과했고 선택형 로컬 Codex fixture 1개는 구성되지 않아 건너뛰었다.
- 2026-07-22: `BehaviorStep.repeatCount`와 프레임 시간 합계 기반 사이클 스케줄러를 추가했다. 일반 행동 변경은 현재 사이클 끝에서 적용하고 단계는 지정한 전체 사이클 횟수 후 다음 단계로 이동한다.
- 2026-07-22: schema-v1 단계는 4단계 마이그레이션 전까지 `LegacyBehaviorStepTiming`으로 원형 보존해 기존 설정의 시간·배속 동작과 저장 왕복을 바꾸지 않는다.
- 2026-07-22: Debug 빌드와 전체 단위 XCTest 157개가 통과했고 선택형 로컬 Codex fixture 1개는 구성되지 않아 건너뛰었다.
- 2026-07-22: schema-v2 설정 저장소를 활성화하고 내장 펫 예약 키와 설치 UUID별 `BehaviorProfile`에 모드, 수동 선택, 루틴과 자동 규칙을 분리했다.
- 2026-07-22: v1 설정은 선택 펫의 실제 애니메이션 사이클로 반복 횟수를 계산한 뒤 v2로 원자적 교체하며, 펫 정의를 얻지 못하거나 변환 저장에 실패하면 원본을 보존하고 쓰기를 차단하도록 연결했다.
- 2026-07-22: 펫 전환과 재실행 뒤 프로필 독립 복원, 다중 프로필 저장 왕복, 손상 프로필 단위 복구와 v1 원본 보호 회귀 테스트를 추가했다.
- 2026-07-22: Debug 빌드, 전체 단위 XCTest 162개와 기존 XCUITest 5개가 통과했고 선택형 로컬 Codex fixture 1개는 구성되지 않아 건너뛰었다.
