## 목표

- 운영체제 API 없이 자동·수동 행동을 결정하는 순수 Domain 엔진을 구현한다.
- 행동 목록을 지정된 시간 동안 진행하고 반복하는 Runtime 스케줄러를 구현한다.
- 전환 우선순위, 유휴 경계, 히스테리시스, 모션 폴백과 상호작용 복귀를 결정적 단위 테스트로 검증한다.

- 수동 모드, 장시간 유휴, 앱 규칙, 짧은 유휴와 기본 행동의 결정 우선순위
- 2분 `rest`, 10분 `sleep` 진입 경계 및 입력 재개 후 3초 이탈 히스테리시스

- 일회성 상호작용 중 기존 목록 일시 정지, 완료 후 남은 시간 복귀, 입력 합치기와 cooldown

- 순수 Domain 및 Runtime 단위 테스트

## 결정사항

- 행동 단계의 `duration`을 최소 유지 구간으로 사용하고 다른 일반 행동 목록은 현재 단계 경계에서 전환한다.
- 상호작용은 즉시 시작하며 완료 후 중단된 단계의 남은 시간으로 복귀한다.
- 유휴 규칙은 10분 이상을 장시간 유휴로 분류해 앱 규칙보다 우선하고, 10분 미만은 앱 규칙 다음에 적용한다.
- 규칙의 `priority` 숫자가 클수록 먼저 평가하고 같은 값은 설정 배열 순서를 유지한다.
- 시간 진행 API는 wall clock을 받지 않고 monotonic clock에서 계산한 `Duration`만 받는다.

## 검증 방법

- 실제 대기 없이 임의의 `ContinuousClock.Instant`와 `Duration`으로 시간 경계를 이동한다.
- 수동 모드와 시스템 일시 중지의 우선순위를 모든 자동 입력과 함께 검증한다.
- 2분·10분 정확한 진입과 입력 재개 후 3초 직전·경계를 검증한다.
- 같은 목록 재요청, 다른 목록 단계 경계 전환, 반복과 비반복 완료를 검증한다.
- 쓰다듬기 완료 후 기존 단계와 남은 시간이 정확히 복구되는지 검증한다.
- 관련 테스트 후 전체 `MonglePetTests`와 Debug 빌드를 실행한다.
