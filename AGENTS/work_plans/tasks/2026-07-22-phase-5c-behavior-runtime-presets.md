# Phase 5C 행동 런타임과 기본 프리셋

## 상태

- 상태: completed
- 생성일: 2026-07-22
- 마지막 갱신: 2026-07-22
- 현재 하위 단계: 완료

## 목표

- 저장된 자동·수동 모드, 행동 목록과 자동 규칙을 `BehaviorResolver`와 `MotionScheduler`를 거쳐 실제 펫 모션에 연결한다.
- 첫 실행에도 자동 휴식·수면과 수동 모션 선택을 검증할 수 있도록 내장 행동 프리셋을 제공한다.
- 설정창에서 수동 모드의 내장 행동을 선택하고 재실행 후 복원한다.

## 범위

- 내장 `idle`, `focus`, `rest`, `sleep` 모션과 기본 행동 목록
- 2분 휴식, 10분 수면 기본 자동 규칙
- 행동 결정, 단계 시간 진행과 실제 `FramePlayer` 연결을 담당하는 `PetBehaviorRuntime`
- 재우기·잠금·절전 중 행동 시간축과 tick 중지
- 행동 단계 `playbackSpeed`의 프레임 재생 속도 적용
- 기본 행동 프리셋의 실행 중 주입과 수동 행동 선택 UI
- 단위 테스트와 실제 앱 수동 QA

## 제외 범위

- 여러 단계 행동 목록 생성·삭제·순서·시간 편집 UI
- 앱별 자동 규칙 생성·삭제 UI
- 유휴 규칙 임계값 편집 UI
- 펫 패키지별 프리셋 자동 생성 정책
- 쓰다듬기 입력과 일회성 상호작용 연결

## 열린 질문

- 없음

## 결정사항

- 첫 실행 또는 행동 목록이 비어 있는 기존 schema-v1 설정에는 내장 프리셋을 메모리에서 주입하고, 다음 사용자 설정 변경 시 함께 저장한다.
- 내장 행동 단계는 3초 경계를 사용해 모드 변경에 대한 최대 대기 시간을 제한한다.
- 자동 기본 규칙은 유휴 2분에 `rest`, 10분에 `sleep`이며 앱별 규칙은 사용자가 다음 단계에서 추가한다.
- 임시 단일 이미지에서는 source rect와 프레임 시간만 다르게 해 상태별 움직임을 구분하고, 정식 펫 자산이 들어오면 동일한 motion ID를 유지한 채 교체한다.
- 행동 tick은 반복 polling이 아니라 현재 단계의 남은 시간에 맞춘 일회성 timer를 사용한다.

## 검증 방법

- 기본 프리셋의 모든 motion ID가 내장 펫 정의에 존재하는지 확인한다.
- 자동 기본·유휴 규칙, 수동 선택과 단계 경계 전환을 가짜 clock과 scheduler로 확인한다.
- 동일 행동 반복은 프레임 재생을 불필요하게 재시작하지 않는지 확인한다.
- playback speed가 프레임 delay에 반영되는지 확인한다.
- 재우기·잠금·절전 중 행동 timer가 취소되고 복귀 후 남은 시간이 유지되는지 확인한다.
- 설정창의 수동 행동 선택이 실제 모션과 설정 파일에 반영되는지 확인한다.
- 전체 `MonglePetTests`, Debug 빌드와 실제 앱 수동 QA를 실행한다.
