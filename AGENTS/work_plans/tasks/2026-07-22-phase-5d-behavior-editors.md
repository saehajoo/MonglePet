# Phase 5D 행동 루틴과 자동 규칙 편집기

## 상태

- 상태: completed
- 생성일: 2026-07-22
- 마지막 갱신: 2026-07-22
- 현재 하위 단계: 완료

## 목표

- 설정 UI에서 여러 단계 행동 루틴을 생성·편집·삭제하고 수동 행동으로 선택할 수 있게 한다.
- 앱 bundle identifier와 유휴 시간에 연결된 자동 규칙을 생성·편집·삭제하고 즉시 runtime에 적용한다.
- 모든 편집이 schema-v1 제한을 만족하고 참조 무결성을 유지하게 한다.

## 범위

- 행동 루틴 생성과 사용자 표시 이름으로 사용하는 고유 ID
- 단계 추가·삭제·순서 변경, motion, 유지 시간과 playback speed 편집
- 반복 여부와 수동 행동 루틴 선택
- 앱 조건·유휴 조건 자동 규칙 생성·편집·활성화·삭제
- 규칙 우선순위 편집과 행동 루틴 참조 선택
- 일반·행동 루틴·자동 규칙 설정 탭
- 같은 ID의 행동 루틴 내용 변경을 다음 단계 경계에 runtime 적용
- 현재 실행 중인 펫이 가진 실제 애니메이션을 단계 선택 항목으로 제공
- 편집 검증 오류 안내와 단위·실제 앱 QA

## 제외 범위

- 행동 루틴 이름 변경
- 설치된 앱 선택 패널과 앱 아이콘 표시
- drag-and-drop 단계·규칙 순서 변경
- 여러 조건 AND/OR 조합
- 펫별 별도 행동 설정
- 쓰다듬기 상호작용

## 결정사항

- schema-v1에 별도 이름 필드가 없으므로 사용자가 입력한 목록 이름을 비어 있지 않은 고유 `id`로 사용하며 이번 단계에서는 이름 변경을 지원하지 않는다.
- `idle`, `focus`, `rest`, `sleep` 내장 목록은 삭제할 수 없지만 단계·반복·속도는 편집할 수 있다.
- custom 목록 삭제 시 해당 목록을 참조하는 자동 규칙은 함께 제거하고 수동 선택은 `idle` 또는 첫 목록으로 복구한다.
- 목록에는 항상 한 단계 이상을 유지하고 단계 시간은 UI에서 1초–24시간, 속도는 0.25–4.0으로 제한한다.
- 앱 bundle identifier는 앞뒤 공백과 내부 공백을 허용하지 않는다. 유휴 임계값은 1–1,440분이다.
- 새 규칙 priority는 현재 최댓값보다 1 크게 만들고 사용자는 Stepper로 조정할 수 있다.
- 같은 ID이지만 내용이 바뀐 행동 목록은 현재 단계의 끝에서 새 내용으로 교체한다.
- UI에서는 저장·Domain의 행동 목록을 `행동 루틴`, 펫 패키지의 motion을 `펫 애니메이션`으로 표시한다.
- 행동 루틴은 다른 루틴을 중첩하지 않고 현재 펫의 애니메이션만 단계로 조합한다.

## 검증 방법

- 중복·빈 이름, 상한 초과와 마지막 단계 삭제를 거부하는지 확인한다.
- 단계 추가·삭제·순서 변경과 목록 삭제 후 참조 정리를 확인한다.
- 앱·유휴 규칙의 조건 값, 우선순위, 활성 상태와 목록 참조 저장을 확인한다.
- 같은 ID 목록 편집이 진행 위치를 즉시 버리지 않고 다음 단계 경계에 적용되는지 확인한다.
- 설정 UI 변경이 JSON에 저장되고 재실행 후 복원되는지 확인한다.
- 앱 규칙과 유휴 규칙이 실제 모션을 선택하는지 확인한다.
- 전체 단위 테스트, Debug 빌드와 실제 앱 수동 QA를 실행한다.

## 완료 결과

- 일반·행동 루틴·자동 규칙 세 탭에서 schema-v1 설정을 안전하게 편집하고 즉시 저장·적용할 수 있다.
- 행동 루틴 단계의 추가·삭제·이동, 애니메이션·유지 시간·재생 속도와 반복 여부를 편집할 수 있다.
- 앱·유휴 규칙의 조건, 대상 루틴, 활성 상태와 우선순위를 편집할 수 있다.
- custom 루틴 삭제 시 수동 선택과 규칙 참조를 정리하며 내장 루틴은 보호한다.
