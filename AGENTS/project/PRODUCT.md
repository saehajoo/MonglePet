# MonglePet 제품 정의

## 제품 개요

MonglePet은 macOS 데스크톱 위에 작은 반려 캐릭터를 표시하고 사용자의 설정과 작업 상황에 맞춰 애니메이션을 전환하는 네이티브 앱이다.

- 제품명: **MonglePet**
- 한글명: **몽글펫**
- 기본 캐릭터명: **몽글이**
- 한 줄 소개: **일할 때도, 쉴 때도 곁에 있는 가벼운 데스크톱 펫**
- 초기 지원 플랫폼: **macOS 14 이상**
- 후속 플랫폼: macOS 버전이 안정화된 뒤 Windows를 별도 네이티브 앱으로 구현

`bssm-oss/desktop-pet`을 비롯한 유사 제품은 기능과 UX를 조사하는 참고 자료로만 사용한다. 소스 코드, 저장소 이력, 내부 구조를 복사하거나 포크하지 않고 새로운 코드베이스로 개발한다.

## 핵심 원칙

### 네이티브와 저전력

- Swift 6, AppKit, SwiftUI, Core Animation을 사용한다.
- 투명 오버레이와 프레임 렌더링은 AppKit/Core Animation으로 구현한다.
- 설정 화면은 SwiftUI로 구현하되 열려 있을 때만 필요한 UI 자원을 유지한다.
- 애니메이션 이미지는 로드 시 디코딩하고 프레임이 변경될 때만 화면을 갱신한다.
- 화면 잠금·절전·펫 숨김 상태에서는 애니메이션 시계를 중지한다.
- Electron, WebView, Unity와 상시 실행용 외부 런타임은 사용하지 않는다.

### 개인정보 보호

자동 행동과 이동에 필요한 다음 정보만 기기 안에서 일시적으로 사용한다.

- 현재 전면 앱의 bundle identifier
- 마지막 키보드 또는 마우스 입력 이후 경과 시간
- 화면 잠금, 절전, 깨우기 상태
- 마우스 따라가기 기능을 켠 동안의 현재 포인터 위치
- 자유 이동 중 현재 목표를 고를 때 필요한 전면 앱 대표 창의 위치와 크기

다음 정보는 읽거나 저장하지 않는다.

- 실제 키 입력 내용
- 창 제목, 문서명, 브라우저 주소
- 화면 캡처와 화면 내용
- 사용자가 작성하는 콘텐츠
- 포인터 위치 기록과 이동 이력
- 전면 앱 창의 위치·크기 기록과 이동 이력

MVP에는 네트워크 기능, 활동 통계, 원격 분석을 넣지 않는다. 앱 규칙과 설정은 로컬에만 저장한다.

### 접근성과 안전

- macOS의 동작 줄이기 설정이 켜져 있으면 정지 프레임 또는 최소 동작을 사용한다.
- 외부 펫 패키지는 이미지와 JSON 데이터만 허용하고 실행 코드, 스크립트, 플러그인을 허용하지 않는다.
- 잘못된 설정은 현재 펫의 기본 애니메이션으로 복구하고, 선택한 펫 패키지가 누락되거나 손상된 경우에만 기본 펫으로 복구한다.

## 주요 사용자 기능

### 펫 표시와 제어

- 몽글이 깨우기와 재우기
- 화면 위 펫 드래그와 크기 조정
- 위치 고정, 마우스 따라가기와 자유 이동 중 하나를 선택하는 펫 이동 모드
- 전체 디스플레이, 선택 모니터 또는 사용자 지정 영역으로 자동 이동 범위 제한
- 항상 위 표시와 선택적 클릭 통과
- 기본 투명도와 클릭 통과 중 마우스가 실제 펫 표시 영역에 겹칠 때의 투명도 조절
- 자동 행동 모드와 수동 행동 모드 전환
- 현재 펫의 애니메이션을 선택해 사용하는 쓰다듬기 같은 짧은 상호작용

### 수동 행동

- 하나의 모션을 고정 재생할 수 있다.
- 여러 애니메이션을 완전한 재생 사이클 단위로 조합해 반복할 수 있다.
- 행동 루틴과 수동 선택은 펫별로 독립적으로 관리한다.
- 사용자가 자동 모드로 돌아가기 전까지 앱 감지와 유휴 시간은 행동을 바꾸지 않는다.

### 자동 행동

- 전면 앱의 bundle identifier에 사용자가 연결한 행동을 재생한다.
- 사용자가 등록한 유휴 시간 규칙에 따라 지정한 행동으로 전환한다.
- 자동 규칙과 연결된 행동 루틴은 펫별로 독립적으로 저장한다.
- 화면 잠금과 시스템 절전 중에는 렌더링을 중지한다.
- 행동 전환의 상세 우선순위는 `../specifications/BEHAVIOR_MODEL.md`에서 관리한다.

### 펫 등록

1. 메뉴에서 **새로운 펫 추가**를 선택한다.
2. `.monglepet`, 지원 spritesheet, GIF/APNG 또는 PNG 시퀀스를 선택한다.
3. 패키지 구조, 파일 크기, 이미지와 모션 목록을 검증한다.
4. 이름, 미리보기, 제작자와 라이선스를 보여준다.
5. 사용자 확인 후 Application Support의 MonglePet 라이브러리로 복사한다.
6. 펫별 기본 행동과 자동 앱 규칙을 설정한다.

사용자가 만든 펫은 이름, 제작자, 버전, 라이선스와 애니메이션을 수정할 수 있다. 가져온 패키지는 원본을 직접 바꾸지 않고 편집 가능한 사본을 만든 뒤 수정한다. 로컬 공유는 데이터 전용 `.monglepet` 내보내기와 가져오기로 제공한다. 후속 공유 프로필에서는 제작자가 선택한 자동·수동 행동 모드, 행동 루틴과 이동 설정을 선택적으로 포함하고, 받는 사용자가 펫만 설치하거나 권장 설정을 적용할 수 있게 한다. 설치 UUID, 화면 좌표, 이동 범위, 투명도와 사용 이력은 공유하지 않는다. 패키지는 선택적인 제작 앱 버전과 최소 호환 앱 버전을 기록할 수 있으며 기존 패키지에는 이 필드를 요구하지 않는다.

MVP에는 로컬 파일 등록만 제공한다. URL 설치, 온라인 카탈로그, 펫 마켓은 서명·해시 검증과 운영 정책을 준비한 뒤 추가한다. 자세한 형식은 `../specifications/PET_PACKAGE.md`에서 관리한다.

## MVP 범위

### 포함

- 투명하고 항상 위에 표시되는 한 개의 펫 오버레이
- 기본 펫 몽글이와 기본 애니메이션 하나
- 펫 드래그, 크기 변경, 클릭 통과, 위치 저장
- 깨우기와 재우기
- 자동 및 수동 모드
- 펫별 행동 루틴과 애니메이션 사이클 기반 반복
- 전면 앱과 유휴 시간 기반 자동 규칙
- 앱별 행동 규칙 등록
- 메뉴 막대 빠른 제어와 SwiftUI 설정창
- `.monglepet` 로컬 패키지 등록
- Codex legacy v1 및 v2 PNG/WebP spritesheet 등록
- 간편 GIF/APNG/PNG 시퀀스 등록
- 펫 정보·기본 애니메이션과 프레임별 재생 시간 편집
- `.monglepet` 내보내기와 가져오기
- 위치 고정, 마우스 따라가기와 자유 이동 모드 및 이동 속도 조절
- 다중 모니터 연결 이동, 자동 이동 범위와 펫 투명도 조절
- 앱 버전 표시와 패키지 최소 호환 버전 확인
- 설정 및 펫 라이브러리 복원
- 동작 줄이기 설정 지원
- 로그인 시 자동 실행 선택

### 제외

- Windows 앱
- 여러 펫 동시 표시
- 온라인 펫 카탈로그와 URL 설치
- 최신 버전 네트워크 조회와 자동 업데이트 설치
- 활동 통계와 클라우드 동기화
- 사운드, 알림, 포모도로
- 충돌·중력·화면 가장자리 보행을 포함한 복잡한 물리 이동
- 패키지 안의 실행 코드 또는 플러그인

## 첫 공개 버전 완료 기준

- macOS에서 설치하고 실행할 수 있다.
- 기본 펫과 사용자가 등록한 펫을 전환할 수 있다.
- 자동·수동 모드와 펫별 행동 목록이 안정적으로 동작한다.
- 사용자가 만든 펫의 정보와 프레임 시간을 수정하고 `.monglepet`으로 교환할 수 있다.
- 자동 이동은 작업을 방해하지 않고 잠금·절전·재우기 중 중지된다.
- 등록 앱과 유휴 시간에 따라 자동 행동이 바뀐다.
- 깨우기·재우기와 쓰다듬기 반응이 동작한다.
- 설정과 펫 라이브러리가 재실행 후 복원된다.
- 키 입력 내용, 창 제목, 화면 내용을 읽거나 저장하지 않는다.
- 성능 기준선과 1시간 메모리 검사가 문서화되어 있다.
- 빌드 방법, 개인정보 설명, 패키지 규격, 라이선스가 문서화되어 있다.
