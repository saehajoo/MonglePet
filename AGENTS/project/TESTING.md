# MonglePet 테스트 전략

## 원칙

- 도메인 규칙은 운영체제 API 없이 빠른 단위 테스트로 검증한다.
- 시스템 API와 파일 시스템은 adapter 경계에서 통합 테스트한다.
- UI 테스트는 핵심 사용자 흐름과 앱 실행 가능 여부에 집중한다.
- 코드 변경 후 가장 좁은 관련 테스트부터 실행하고 영향 범위에 따라 검증을 확장한다.
- 시간 기반 로직은 실제 대기 대신 주입 가능한 시계로 검증한다.

## 기본 빌드

```sh
xcodebuild -project MonglePet.xcodeproj \
  -scheme MonglePet \
  -configuration Debug \
  -destination 'platform=macOS' \
  -derivedDataPath /tmp/MonglePetDerivedData \
  CODE_SIGNING_ALLOWED=NO \
  build
```

## 단위 테스트

```sh
xcodebuild -project MonglePet.xcodeproj \
  -scheme MonglePet \
  -configuration Debug \
  -destination 'platform=macOS' \
  -derivedDataPath /tmp/MonglePetDerivedData \
  CODE_SIGNING_ALLOWED=NO \
  test -only-testing:MonglePetTests
```

주요 단위 테스트 대상:

- 자동·수동 모드 우선순위
- 유휴 시간 경계값과 히스테리시스
- 앱 규칙 매칭과 기본 폴백
- 행동 목록 시간 진행과 반복
- Domain `Duration`과 저장 DTO 정수 밀리초 변환
- 명시적인 자동 규칙 조건 JSON discriminator
- 현재 펫 기본 참조와 누락된 애니메이션의 선언된 기본 애니메이션 대체
- 패키지 경로, 크기, 이미지와 manifest 검증
- Codex legacy v1 8×9 및 v2 8×11 WebP atlas 매핑
- 정적 WebP와 animated WebP 판별 및 미지원 형식 거부
- 손상된 설정 복구와 스키마 마이그레이션

## UI 테스트

```sh
xcodebuild -project MonglePet.xcodeproj \
  -scheme MonglePet \
  -configuration Debug \
  -destination 'platform=macOS' \
  -derivedDataPath /tmp/MonglePetDerivedData \
  test -only-testing:MonglePetUITests
```

UI 테스트는 앱 실행과 접근성 자동화가 가능한 macOS 세션에서 실행한다. 기본 스모크 테스트는 앱 실행, 메뉴 막대 항목 또는 펫 창 노출, 핵심 설정 화면 진입 여부를 확인한다.

## 통합 테스트

- 펫 설치와 재실행 후 복원
- 전면 앱 전환에 따른 행동 변경
- 사용자 유휴 규칙 진입과 입력 재개 후 히스테리시스 해제
- 화면 잠금·절전에서 렌더링 중지
- 설정 변경의 즉시 반영
- 펫 제거 시 파일과 설정 정리
- 원자적 설정 저장과 손상 파일 복구

## 수동 QA

- 다중 모니터와 화면 배율 변경
- 전체 화면 앱과 여러 Space
- 클릭 통과와 드래그
- 로그인 직후 실행
- 동작 줄이기 설정
- 매우 크거나 잘못된 외부 패키지
- lossless/lossy WebP와 알파가 없는 WebP 가져오기 결과
- 장시간 실행과 절전 복귀

## 성능 검증

- 펫 숨김과 화면 잠금 상태에서 애니메이션 시계가 중지되는지 확인한다.
- 일반 애니메이션 평균 CPU 1% 미만을 초기 목표로 측정한다.
- 설정창을 닫은 뒤 관련 UI 메모리가 해제되는지 확인한다.
- 1시간 실행 후 지속적인 메모리 증가가 없는지 확인한다.
- 이미지 디코딩과 프레임 캐시의 최대 메모리 사용량을 큰 테스트 패키지로 측정한다.

### Phase 1 기준선

- 측정일: 2026-07-21
- 대상: Debug 빌드, 정적 펫 표시 및 사용자 입력이 없는 상태
- 방법: `top`으로 1초 간격 5회 측정
- 결과: CPU 5회 모두 0.0%, 메모리 약 26MB
- 비고: 애니메이션 런타임이 추가되는 Phase 2에서 Instruments로 다시 측정한다.

### Phase 2B 패키지 로더 검증

- 측정일: 2026-07-21
- 실제 PNG 및 정적 WebP fixture로 이미지 형식, 픽셀 크기와 알파 판별 확인
- 경로 탈출, 심볼릭 링크, 스크립트, 누락·중복 참조와 자원 상한 거부 확인
- 패키지 로더 테스트 15개 및 전체 단위 테스트 38개 통과
- 알파 포함 lossless/lossy WebP와 Codex atlas fixture는 Phase 2D에서 추가 검증

### Phase 2C 패키지 설치 검증

- 측정일: 2026-07-21
- 실제 ZIP 생성 후 security-scoped 접근, 임시 추출, 재검증과 UUID 라이브러리 설치 확인
- 경로 탈출, 절대 경로, 심볼릭 링크, 경로 충돌, 손상 ZIP과 압축 폭탄 제한 확인
- 중복 거부, 별도 사본, 같은 패키지 ID 교체와 실패 시 workspace·staging 정리 확인
- 패키지 설치 테스트 10개 및 전체 단위 테스트 48개 통과

### Phase 2D 호환 가져오기 검증

- 측정일: 2026-07-21
- 생성한 1536×1872 v1 및 1536×2288 v2 atlas로 Codex 행, 시간과 16방향 매핑 확인
- 비어 있는 사용 셀, 불투명한 미사용 셀, version/크기 불일치와 경로 탈출 거부 확인
- 실제 로컬 Codex legacy WebP 펫을 저장소에 복사하지 않고 변환 및 재검증 성공
- ImageIO로 만든 GIF/APNG의 프레임 시간, 투명 GIF canvas 합성과 100ms 지연 시간 폴백 확인
- PNG 시퀀스가 호출자 지정 순서를 보존하고 모든 결과가 기존 패키지 로더를 통과하는지 확인
- 전체 단위 테스트 62개 통과, 환경 변수로 실행하는 로컬 WebP 수동 fixture 테스트 1개는 기본 실행에서 skip

### Phase 3 행동 엔진 검증

- 측정일: 2026-07-22
- `tuckedAway`, 잠금·절전, 상호작용, 수동, 장시간 유휴, 앱 규칙, 짧은 유휴와 기본 행동의 우선순위 확인
- 2분·10분 진입 경계와 입력 재개 후 3초 직전·정확한 이탈 경계 확인
- 높은 `priority` 우선과 같은 값의 안정적인 배열 순서 확인
- 반복·비반복 목록, 같은 목록 진행 보존, 다른 목록의 단계 경계 전환과 pause·resume 확인
- 쓰다듬기 입력 합치기, 기존 단계의 남은 시간 복귀, 500ms cooldown 확인
- 누락 모션의 `idle` 폴백과 `idle`도 없는 펫의 실행 불가 상태 확인
- 행동 엔진 테스트 14개 및 전체 단위 테스트 76개 통과, Debug 빌드 성공

### Phase 4 macOS 활동 감지 검증

- 측정일: 2026-07-22
- 격리된 NotificationCenter로 전면 앱 bundle identifier 변경과 세션·화면·전원 상태 변환 확인
- 가짜 clock, idle provider와 poller로 초기 snapshot, 1초 갱신과 앱 변경 즉시 발행 확인
- 사용자 세션 비활성화·화면 수면·시스템 절전 중 polling 중지 및 복귀 후 유휴 시간 재조회 확인
- 실제 `CGEventSource` 유휴 시간 조회가 별도 권한 설정 없이 테스트 호스트에서 동작하는지 확인
- 시스템 일시 중지가 사용자의 awake/tuckedAway 상태를 바꾸지 않고 애니메이션만 중지하는지 확인
- 활동 감지 관련 테스트 8개 및 전체 단위 테스트 84개 통과, Debug 빌드 성공
- 실제 앱에서 별도 권한 요청 없이 잠금·해제 및 절전·깨우기 후 애니메이션 재개 확인
- 사용자 재우기 상태에서 잠금·해제 후 계속 숨김을 유지하고 직접 깨울 때만 재개하는지 확인

### Phase 5A 설정 저장과 복원 기반 검증

- 측정일: 2026-07-22
- 기본값 로드와 행동 목록·자동 규칙·overlay를 포함한 schema-v1 저장 후 재로드 결과 확인
- 정수 밀리초, 명시적 enum 문자열과 `application`·`idleAtLeast` 조건 discriminator 확인
- 잘못된 필드·UUID·목록 참조의 항목 단위 복구와 알 수 없는 조건 type 보존·비활성화 확인
- 손상·5MiB 초과 파일 격리, 미래 스키마 원본 보존과 쓰기 차단 확인
- 기존 파일 원자적 교체, 실패 시 임시 파일 정리와 유효하지 않은 Domain 설정 저장 거부 확인
- 설정 저장 테스트 7개와 미지원 자동 규칙 무시 테스트 1개 추가, 전체 단위 테스트 92개 통과, Debug 빌드 성공
- 이 단계는 앱 런타임과 설정 UI에 연결하지 않았으므로 별도 수동 QA 대상은 없음

- 설정 세션의 즉시 변경, 명시적 저장, 오류 안내와 미래 스키마 원본 보존 확인
- 첫 실행 우하단 배치, 저장 위치 복원, 화면 밖 보정과 적용 좌표의 메모리 동기화 확인
- 192:208 종횡비 크기 변경과 `NSPanel.ignoresMouseEvents` 클릭 통과 적용 확인
- 저장된 `awake`·`tuckedAway`, 행동 모드, 크기, 클릭 통과와 위치의 재실행 복원 확인
- 전체 단위 테스트 99개 통과, 환경 변수 기반 로컬 WebP fixture 테스트 1개는 기본 실행에서 skip, Debug 빌드 성공

### Phase 5C 행동 런타임과 기본 프리셋 검증

- 측정일: 2026-07-22
- 내장 idle·focus·rest·sleep 모션의 atlas 범위와 프리셋 motion ID 일치 확인
- 가짜 clock과 일회성 scheduler로 수동 목록의 3초 단계 경계 전환과 자동 2분·10분 유휴 규칙 확인
- 재우기·잠금·절전 중 timer 취소, 남은 단계 시간 보존과 복귀 후 재예약 확인
- `playbackSpeed`에 따른 프레임 delay와 잘못된 속도의 1.0배 복구 확인
- 전체 단위 테스트 105개 통과, 환경 변수 기반 로컬 WebP fixture 테스트 1개는 기본 실행에서 skip, Debug 빌드 성공
- 실제 앱에서 수동 4개 프리셋 전환, 수동 선택·모드 재실행 복원과 잠금 해제 후 애니메이션 재개 확인
- 실제 2분 유휴 대기 QA는 생략했으며 같은 경계는 가짜 시계 단위 테스트로 검증

### 사용자 자유 행동 기본값 전환 검증

- 측정일: 2026-07-22
- 첫 실행 설정을 단일 `기본` 루틴과 자동 규칙 0개로 정규화하는지 확인
- 기본 루틴과 새 사용자 루틴이 예약된 현재 펫 기본 애니메이션 참조로 시작하는지 확인
- 앱·유휴 조건이 동시에 일치할 때 조건 종류가 아닌 큰 `priority`와 안정적인 배열 순서로 평가하는지 확인
- `idle` 애니메이션이 없는 펫에서도 선언된 기본 애니메이션을 재생하고 정상 참조를 오류 폴백으로 표시하지 않는지 확인
- 정확히 일치하는 구형 4개 루틴·2개 규칙은 새 기본값으로 전환하고 수정된 구형 데이터는 보존하는지 확인
- 전체 XCTest 136개 통과, 선택적 로컬 WebP fixture 1개 건너뜀
- 단일 기본 루틴 편집, 사용자 루틴·자동 규칙 구성과 재실행 복원 실제 앱 QA 통과

### Phase 6 펫 상세와 애니메이션 미리보기 검증

- 측정일: 2026-07-22
- overlay와 설정 미리보기가 내장·설치 펫에 동일한 atlas 로더를 사용하는지 확인
- 설정 미리보기에서는 창 드래그를 비활성화하고 선택 애니메이션의 실제 프레임 재생을 기존 `FramePlayer`로 연결
- 현재 펫의 전체 애니메이션 목록, 기본 표시, 프레임 수·사이클 시간·반복 요약을 설정 UI에 제공
- 전체 단위 XCTest 137개 통과, 선택적 로컬 WebP fixture 1개 건너뜀
- 설정창에서 미리보기와 애니메이션 목록 노출을 확인하는 대상 XCUITest 1개 통과
- 실제 앱에서 애니메이션 선택 전환, 표시 정보와 펫 교체 시 기본 애니메이션 복귀 QA 통과

### Phase 6 사용자 펫 메타데이터 편집 검증

- 측정일: 2026-07-22
- 새 사용자 펫의 이름·제작자·버전·라이선스·설명을 manifest에 정규화해 저장하는지 확인
- 기존 사용자 펫의 메타데이터와 기본 애니메이션을 임시 사본에서 수정하고 같은 설치 UUID로 원자적 교체하는지 확인
- 패키지 ID, 애니메이션과 편집 marker가 유지되고 저장 후 라이브러리 선택과 런타임에 변경 정의를 다시 전달하는지 확인
- 존재하지 않는 기본 애니메이션과 읽기 전용 가져온 펫의 수정을 거부하고 기존 manifest를 변경하지 않는지 확인
- 전체 단위 XCTest 141개 통과, 선택적 로컬 WebP fixture 1개 건너뜀
- 설정창 기본 스모크와 새 펫 메타데이터 입력란을 확인하는 관련 XCUITest 2개 통과
- 실제 앱에서 새 펫 메타데이터 입력, 정보 수정, 기본 애니메이션 전환, 즉시 반영, 재실행 복원과 읽기 전용 UI QA 통과

### UI 자동화 환경 기록

- 설정창 UI 스모크 테스트는 Phase 1A에서 통과했다.
- Phase 1D 전체 UI 테스트 재실행은 Xcode Runner가 앱과 연결되기 전에 signal kill로 종료되어 완료하지 못했다.
- Phase 5B 설정창 UI 테스트는 test runner가 macOS automation mode를 활성화하는 단계에서 75초 후 timeout되어 완료하지 못했다.
