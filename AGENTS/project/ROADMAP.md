## 1단계: 화면에 보이는 최소 제품

### 1A: 앱 셸

- 기본 SwiftUI `WindowGroup` 제거
- `AppCoordinator` 기반 앱 생명주기 구성
- AppKit `NSStatusItem` 메뉴 막대 항목 추가
- Dock 아이콘을 표시하지 않는 agent-style 앱 설정
- 설정 열기와 앱 종료 진입점 추가

완료 조건: 일반 앱 창이나 Dock 아이콘 없이 메뉴 막대에서 MonglePet을 제어하고 종료할 수 있다.

### 1B: 펫 오버레이

- 투명하고 테두리가 없는 non-activating `NSPanel`
- 교체 가능한 임시 투명 PNG 표시
- 일반 앱 위, 모든 Space와 전체 화면 보조 공간에 표시
- 화면 보호기 및 잠금 화면보다 위에는 표시하지 않음

완료 조건: 현재 작업 중인 앱의 활성 상태를 빼앗지 않고 임시 몽글이가 안정적으로 표시된다.

### 1C: 기본 상호작용

- 펫 드래그
- 깨우기와 재우기
- 디스플레이 구성 변경 시 화면 밖 위치 보정
- 메뉴 막대에서 숨겨진 펫을 다시 깨울 수 있는 복구 경로 유지

완료 조건: 임시 몽글이를 이동하고 숨겼다가 메뉴 막대에서 다시 표시할 수 있다.

### 1D: 검증

- 표시 상태 전환 단위 테스트
- 메뉴 막대 및 앱 실행 UI 스모크 테스트
- 다중 모니터, 전체 화면과 여러 Space 수동 확인
- 정지 상태 CPU 기준선 측정 및 기록

단계 완료 조건: 임시 몽글이 이미지가 안정적으로 표시·이동·숨김 처리되고 현재 작업 앱을 방해하지 않는다.

- [ ] 편집 marker, 행동 루틴과 자동 규칙 제외
- [ ] 내보내기 전 전체 패키지 재검증
- [ ] 가져오기, 중복 설치, 같은 설치 업데이트 UX 정리
- [ ] 공유받은 펫의 읽기 전용 설치와 편집 가능한 사본 만들기

완료 조건: 두 사용자가 실행 코드와 개인 행동 설정 없이 같은 펫 패키지를 안전하게 교환할 수 있다.

## 9단계: 이동과 상호작용

- [ ] 마우스 따라가기 켜기·끄기
- [ ] 이동 속도, 커서와의 거리와 정지 반경 설정
- [ ] 사용자 드래그 중 자동 이동 일시 중지
- [ ] 다중 모니터, 화면 경계, 클릭 통과와 전체 화면 처리
- [ ] 재우기·잠금·절전·동작 줄이기 중 이동 중지
- [ ] 이동 중 애니메이션과 기존 행동 복귀
- [ ] 쓰다듬기 일회성 반응

완료 조건: 선택적 이동과 상호작용이 현재 작업을 가로막지 않고 행동 엔진과 독립적으로 동작한다.

## 10단계: 성능과 배포 안정화

- Instruments 기준선 측정
- 화면 잠금과 재우기 상태의 렌더링·이동 중지 확인
- 1시간 이상 메모리 증가 검사
- 다중 모니터, Retina, 전체 화면, 여러 Space QA
- 로그인 시 실행, 앱 서명, notarization 검토
- 개인정보 및 오픈소스 고지 작성

초기 목표:

- 펫 숨김·정지 상태 평균 CPU: 측정 오차에 가까운 수준
- 일반 애니메이션 평균 CPU: 1% 미만 목표
- 마우스 따라가기 비활성 상태에서 이동 기능의 상시 polling 없음
- 설정창을 닫은 뒤 관련 UI 메모리 해제
- 1시간 실행 후 지속적인 메모리 증가 없음

## 11단계: Windows 별도 구현

macOS에서 검증된 펫 패키지, 설정 스키마와 테스트 시나리오만 공유한다. Windows UI와 시스템 감지, 렌더러 및 이동 컨트롤러는 별도 네이티브 앱으로 구현한다.
